## 1. 문제 정의

### 대부분의 직장인은 자기 경력을 모른다

- 퇴직 직전에 이력서를 쓰려 하면 "3년 동안 내가 뭘 했지?"가 된다
- 업무는 매일 하지만 기록이 없다
- 이직 결심 후 2주 안에 경력을 정리하려다 포기한다
- 결국 직함과 회사명만 쓴 "빈 이력서"로 지원한다

### 기존 솔루션의 한계

| 도구 | 문제 |
|------|------|
| 노션/옵시디언 | 기록은 되지만 구조화·자동화 없음. 결국 안 씀 |
| 링크드인 | 본인이 직접 써야 함. 업데이트 귀찮아서 방치 |
| AI 이력서 툴 | 작성 도움만. 기억 자체가 없으면 소용없음 |
| 일반 메모앱 | 경력 맥락 없이 흩어짐 |

### 핵심 인사이트

> 문제는 "이력서 작성 능력"이 아니라 **"기록되지 않은 경험"** 이다.

---

## 2. 솔루션

### CareerLog — 일하면서 자동으로 쌓이는 경력 저장소

대화로 오늘 한 일을 말하면, AI가 알아서 구조화해서 저장한다.
쌓인 기록은 언제든 이력서·경력기술서·LinkedIn으로 꺼낼 수 있다.

```
오늘 업무 입력 (대화/음성/파일)
        ↓
AI가 분류·구조화·저장
        ↓
프로젝트별 / 직무별 / 연도별 경력 누적
        ↓
이직 시 → 이력서·경력기술서 자동 생성
         → 지원 전략·피드백 제공
```

## 3. 핵심 기능

### Phase 1 — 기록 (MVP)

| 기능 | 설명 |
|------|------|
| **대화형 업무 기록** | "오늘 이런 거 했어"를 말하면 AI가 구조화해서 저장 |
| **파일 분석 저장** | PDF·PPT·Figma 등 업무 산출물을 분석해서 경력에 통합 |
| **자동 분류** | 프로젝트명·직무 카테고리·날짜 자동 태깅 |
| **수치 감지** | "오류 75% 감소" 같은 수치 자동 하이라이트 및 성과로 저장 |

### Phase 2 — 조회·관리

| 기능 | 설명 |
|------|------|
| **타임라인 뷰** | 연도별·회사별·프로젝트별 경력 시각화 |
| **성과 대시보드** | 저장된 수치 성과 모아보기 |
| **검색** | "내가 Figma 관련해서 한 게 뭐가 있지?" 자연어 검색 |

### Phase 3 — 출력

| 기능 | 설명 |
|------|------|
| **이력서 자동 생성** | 쌓인 경력 기반 이력서 초안 자동 생성 |
| **경력기술서 자동 생성** | 문제→접근→결과 스토리 구조로 자동 작성 |
| **LinkedIn 최적화** | 섹션별 최적화 텍스트 생성 |
| **JD 매칭** | 채용공고 붙여넣으면 내 경력과 매칭 분석 |

### Phase 4 — 피드백·전략

| 기능 | 설명 |
|------|------|
| **현직자 페르소나 심사** | 목표 회사 직군 팀장 인격으로 서류 심사·피드백 |
| **모의 면접** | 저장된 경력 기반 예상 질문 + 답변 코칭 |
| **이직 전략 리포트** | 현재 경력 갭 분석 + 보완 로드맵 제안 |

## 4. 타겟 유저

### 메인 타겟

**직장인 3~7년차 이직 준비자**
- 경력은 있는데 정리가 안 된 사람
- 이직 결심했지만 이력서 쓰기 막막한 사람
- 포트폴리오가 없는 기획자·PM·디자이너

### 세컨더리 타겟

- **신입 취준생** — 인턴·프로젝트·수업 경험 기록
- **프리랜서** — 다수 클라이언트 프로젝트 관리
- **HR 담당자 (B2B)** — 직원 성과 기록 + 인사 연동

### 페르소나 예시

| 이름 | 상황 |
|------|------|
| 백지민 (30, Product Designer) | 다이버스에서 3년, 이직 준비 중. 한 게 많은데 정리가 안 됨 |
| 변지현 (32, 게임기획) | QA·기획·PM 멀티롤. 뭘 어필해야 할지 모름 |
| 이민준 (28, 개발자) | 사이드 프로젝트 많은데 이력서에 어떻게 쓸지 모름 |

## 6. 경쟁사 분석

| 서비스 | 강점 | 우리와의 차이 |
|--------|------|--------------|
| Resume.io / Rezi | AI 이력서 작성 | 작성 도움만, 기억 저장 없음 |
| 사람인 / 잡코리아 | 국내 채용 플랫폼 강점 | 기록·축적 기능 없음 |
| LinkedIn | 글로벌 네트워크 | 본인이 직접 써야 함 |
| 노션 | 자유로운 기록 | 경력 특화 구조·AI 없음 |
| **CareerLog** | **일상 기록 → 경력 자동 축적** | **유일한 포지션** |

### 핵심 차별화

> 경쟁사들은 모두 **"이미 기억하고 있는 것"을 잘 쓰게 도와준다.**
> CareerLog는 **"기억할 필요 없게"** 만든다.

| **② 에이전트 파이프라인** | career-recorder → resume-writer → job-analyzer | 사용자 입력을 단계별로 처리·저장·출력하는 멀티 에이전트 워크플로우 | ✅ Harry 프로젝트에서 파이프라인 구동 확인 |
| **③ 경력 저장·검색** | PostgreSQL + Vector DB (pgvector 또는 Pinecone) | 구조화된 경력 데이터 저장 + 자연어 기반 시맨틱 검색 | 🔲 Pre-MVP에서는 Git 기반 파일 저장으로 대체 중 |

#### ① AI 엔진 — Claude API 연동 상세

```
[사용자 입력]
  "오늘 디자인 시스템 v2 컴포넌트 라이브러리 정리 완료. 
   기존 대비 컴포넌트 수 40% 줄이고 일관성 높임"
            │
            ▼
  ┌─────────────────────────────┐
  │  Claude API — Structured    │
  │  Output (JSON Schema)       │
  │                             │
  │  • 프로젝트명 추출: 디자인 시스템 v2  │
  │  • 직무 카테고리: Product Design   │
  │  • 수치 성과 감지: 컴포넌트 40% 감소 │
  │  • STAR 프레임 매핑:              │
  │    S: 기존 라이브러리 비효율         │
  │    T: 컴포넌트 정리·통합           │
  │    A: v2 리뉴얼 주도              │
  │    R: 40% 경량화, 일관성 향상      │
  │  • 날짜 태깅: 2026-03-17         │
  └─────────────────────────────┘
            │
            ▼
  [경력 저장소에 구조화된 레코드로 저장]
```

**프롬프트 엔지니어링 전략:**

- **시스템 프롬프트 분리 관리** — 기록(record), 생성(write), 분석(analyze) 각 에이전트별 전용 시스템 프롬프트를 독립적으로 버전 관리
- **Few-shot 예시 기반 품질 확보** — 직무별(기획, 디자인, 개발, 마케팅 등) 구조화 예시를 프롬프트에 포함하여 출력 일관성 유지
- **Structured Output 강제** — Claude API의 JSON Schema 모드를 활용하여 항상 동일한 데이터 구조로 응답받음
- **비용 최적화** — 기록 분류는 Claude Haiku(저비용 모델), 이력서 생성·페르소나 심사는 Claude Sonnet(고품질 모델)로 모델을 분리하여 API 비용 관리 (목표: 유저당 월 평균 API 비용 500원 이하)

#### ③ Vector DB — 경력 시맨틱 검색

CareerLog의 핵심 사용자 경험 중 하나인 **"내가 Figma 관련해서 한 게 뭐가 있지?"** 같은 자연어 검색을 가능하게 하는 기술이다.

**구현 계획:**

```
[경력 레코드 저장 시]
  구조화된 텍스트 → Claude Embedding API로 벡터 변환 → Vector DB에 저장

[자연어 검색 시]
  사용자 질의 → 벡터 변환 → 코사인 유사도 기반 Top-K 검색 
  → 관련 경력 레코드 반환 → 맥락과 함께 LLM이 요약·정리하여 응답
```

| 구분 | 1차 선택 | 대안 | 선택 이유 |
|------|----------|------|-----------|
| Vector DB | **pgvector** (PostgreSQL 확장) | Pinecone, Weaviate | 별도 인프라 없이 기존 PostgreSQL과 통합 가능. 초기 규모에서 충분한 성능. 비용 절감 |
| Embedding 모델 | Claude Embedding 또는 OpenAI text-embedding-3-small | Cohere Embed | Claude API 생태계 내 일관성 유지 |

### 기술 차별성 — 기존 이력서 툴 대비

| 비교 항목 | 기존 AI 이력서 툴 (Resume.io, Rezi 등) | CareerLog |
|-----------|----------------------------------------|-----------|
| **데이터 원천** | 사용자가 기억에 의존하여 직접 입력 | 일상적 업무 기록이 자동으로 누적 → 기억에 의존하지 않음 |
| **AI 활용 범위** | 이력서 문장 다듬기, 포맷팅 | 업무 분류·구조화·수치 감지·성과 추출·이력서 생성·JD 매칭·페르소나 심사까지 **경력 관리 전 과정** |
| **데이터 축적** | 1회성 — 작성 후 끝 | 지속적 축적 — 기록할수록 풍부해지는 **경력 자산** |
| **개인화 수준** | 템플릿 기반 범용 출력 | 축적된 실제 업무 데이터 기반 **고도로 개인화된 출력** |
| **네트워크 효과** | 없음 | 사용 기간이 길수록 데이터 밀도 증가 → 출력 품질 향상 → **록인 효과** |
| **B2B 확장성** | 제한적 (개인 도구) | 직원 성과 기록 → HR 시스템 연동 → **기업 단위 도입 가능** |

**핵심 기술 차별 요약:**

> 기존 도구는 **"빈 종이 위에서 시작"** 한다. CareerLog는 **"이미 쌓여 있는 데이터에서 시작"** 한다. 이 차이가 출력 품질, 사용자 경험, 록인 효과 모두를 결정한다.

#### AI 출력 품질 관리 — 상세

CareerLog에서 AI 출력 품질은 곧 서비스 품질이므로, 별도의 관리 프로세스를 운영한다.

```

### 사업화 전략 개요

CareerLog의 사업화는 **"좁고 깊게 시작하여, 넓고 빠르게 확장한다"**는 원칙을 따른다. Pre-MVP 단계의 CLI 프로토타입으로 핵심 가치를 검증한 뒤, 웹 서비스 출시와 함께 B2C 구독 매출을 확보하고, PMF 검증 이후 B2B 파트너십과 해외 진출로 성장을 가속하는 3단계 구조다.

```
[사업화 3단계 구조]

1단계 (1~6개월)     2단계 (7~18개월)      3단계 (19~36개월)
커뮤니티 기반        퍼포먼스 마케팅 +       B2B 파트너십 +
오가닉 성장          콘텐츠 SEO 확대        해외 시장 진입
                                          
베타 10명 →          유료 1,000명 →         유료 15,000명
MVP 유저 100명       B2C 안정화             B2B + 글로벌
```

### 1. 초기 고객 획득 전략 (베타 유저 10명 → 유료 전환)

#### Phase 0: 씨앗 유저 확보 (Pre-MVP, 1~4주)

초기 10명의 베타 유저는 **제품의 핵심 가치를 가장 절실히 느낄 수 있는 사람**으로 선별한다. 광고가 아닌 직접 대화와 관계 기반으로 모집한다.

| 순서 | 활동 | 목표 | 채널 |
|------|------|------|------|
| 1 | 창업자 직접 네트워크에서 이직 준비 중인 지인 선별 | 5명 | 1:1 메시지 (카카오톡, 슬랙 DM) |
| 2 | 이직·커리어 커뮤니티에 문제 공감 게시글 작성 후 관심자 모집 | 3명 | 블라인드, 커리어리, 디스콰이엇 |
| 3 | 기획자·PM·디자이너 오픈 채팅방에서 1:1 인터뷰 제안 | 2명 | 오픈카카오, 슬랙 커뮤니티 |

**선별 기준:**

- 직장인 3~7년차, 이직을 구체적으로 고려 중인 사람
- "이력서 쓰기 막막하다"는 문제를 실제로 겪고 있는 사람
- CLI 도구에 대한 거부감이 낮은 사람 (초기 프로토타입이 CLI이므로)
- 피드백을 적극적으로 줄 의향이 있는 사람

**베타 유저 운영 방식:**

```
[베타 유저 여정]

1주차: 온보딩 인터뷰 (30분) — 현재 경력 상태, 이직 목표 파악
       → 초기 경력 정보 CLI에 입력 대행
       
2주차: 매일 업무 기록 독려 — 슬랙 DM으로 "오늘 뭐 하셨어요?" 리마인드
       → 기록 입력 → AI 구조화 결과 확인 → 즉시 피드백

3주차: 이력서·경력기술서 생성 기능 체험
       → "이 정도면 돈 낼 의향 있나요?" 핵심 질문

4주차: 종합 피드백 수집 — 사용 빈도, 가장 유용한 기능, 불편 사항
       → NPS + 유료 전환 의향 설문
```

**Phase 0 전환 시나리오:**

| 지표 | 목표 | 판단 기준 |
|------|------|-----------|
| 주 3회 이상 기록 유저 비율 | 50% (5/10명) | 50% 이상 → 핵심 습관 형성 가능 확인 |

### 4. 고객 획득 비용 (CAC) 및 생애 가치 (LTV) 추정

#### CAC (Customer Acquisition Cost) 추정

| 구분 | 1년차 (추정) | 2년차 (추정) | 3년차 (추정) |
|------|-------------|-------------|-------------|
| 마케팅 총 비용 | 약 600만 원 | 약 6,000만 원 | 약 1억 4,000만 원 |
| 신규 유료 전환자 수 | 약 1,000명 | 약 4,500명 (순증) | 약 12,000명 (순증) |
| **유료 전환 기준 CAC** | **약 6,000원** | **약 13,300원** | **약 11,700원** |

> 1년차 CAC가 낮은 이유: 오가닉(커뮤니티·콘텐츠) 중심 성장으로 유료 마케팅 비용이 최소화되기 때문. 2년차 이후 퍼포먼스 마케팅 확대로 CAC 상승하나, 규모의 경제와 브랜드 인지도 향상으로 3년차에는 다시 감소 추세 목표

#### LTV (Lifetime Value) 추정

```
[LTV 산출]

LTV = ARPU × 평균 구독 유지 기간 (월)

ARPU: 약 12,000원/월 (Pro 60% + Premium 30% + Team 10% 가중 평균)

월 이탈률별 평균 구독 유지 기간:
 - 이탈률 12% (1년차): 1/0.12 ≈ 8.3개월
 - 이탈률 8% (2년차):  1/0.08 = 12.5개월
 - 이탈률 5% (3년차):  1/0.05 = 20개월

LTV 추정:
 - 1년차 기준: 12,000원 × 8.3개월 ≈ 약 100,000원
 - 2년차 기준: 12,000원 × 12.5개월 ≈ 약 150,000원
 - 3년차 기준: 12,000원 × 20개월 ≈ 약 240,000원
```

| 구분 | 1년차 (추정) | 2년차 (추정) | 3년차 (추정) |
|------|-------------|-------------|-------------|
| **ARPU** | 약 10,500원/월 | 약 12,000원/월 | 약 12,000원/월 |
| **평균 구독 유지 기간** | 약 8.3개월 | 약 12.5개월 | 약 20개월 |
| **LTV** | **약 87,000원** | **약 150,000원** | **약 240,000원** |
| **CAC** | **약 6,000원** | **약 13,300원** | **약 11,700원** |
| **LTV / CAC 비율** | **약 14.5x** | **약 11.3x** | **약 20.5x** |

> **LTV/CAC 비율이 전 기간에 걸쳐 3x 이상**을 유지하며, 이는 SaaS 비즈니스의 건전성 기준(일반적으로 LTV/CAC > 3x가 양호)을 충분히 상회한다.

**LTV 향상 전략:**

| 전략 | 효과 | 시기 |
|------|------|------|
| **기록 누적 록인 효과** | 사용 기간이 길수록 경력 데이터 밀도 증가 → 이탈 비용 상승 | 상시 |

### 사회적 가치 개요

CareerLog는 단순한 생산성 도구가 아니라, **한국 노동 시장의 구조적 비효율을 해소하는 사회적 인프라**를 지향한다. "일한 만큼 기록되고, 기록된 만큼 인정받는" 환경을 만듦으로써 개인의 직업 이동성을 높이고, 기업의 채용 미스매치를 줄이며, 노동 시장 전체의 효율성을 향상시키는 것이 CareerLog의 사회적 미션이다.

---

### 1. 해결하는 사회 문제 — 기록되지 않는 경력, 멈추는 커리어

#### 문제의 구조

한국 직장인의 경력은 **"회사명 + 직함 + 재직기간"** 으로만 표현되는 것이 현실이다. 3년, 5년, 10년 동안 수행한 실제 업무·성과·역량은 어디에도 체계적으로 기록되지 않는다. 이로 인해 다음과 같은 사회적 비용이 발생한다.

| 문제 | 영향 | 규모 (추정) |
|------|------|------------|
| **이력서 공백** | 실제 역량 대비 이력서 표현력이 낮아 지원 자체를 포기하거나, 역량에 못 미치는 포지션에 지원 | 연간 이직자 약 260만 명 중 상당수가 경력 정리 난항 경험 (추정) |
| **직업 이동성 저하** | 자신의 경력을 설명하지 못해 이직·전직을 미루거나 포기 → 부적합한 직무에 장기 체류 | MZ세대 평균 재직기간 2.4년이지만, 이직 준비 기간은 평균 3~6개월 소요 (추정) |
| **채용 미스매치** | 기업은 후보자의 실제 역량을 파악하기 어렵고, 후보자는 자신에게 맞는 공고를 찾기 어려움 | 채용 후 1년 내 퇴사율 약 30% (추정, 사람인 2024 조사 참고) — 미스매치가 주요 원인 |
| **경력단절의 고착화** | 육아·돌봄 등으로 경력이 단절된 이후, 이전 경력을 증명할 수단이 없어 복귀가 더욱 어려워짐 | 경력단절 여성 약 144만 명 (통계청, 2024) |

#### CareerLog가 만드는 변화

> **"기억에 의존하던 경력"을 "데이터로 축적된 경력"으로 전환**함으로써, 누구나 자신이 한 일을 정확히 설명하고, 적합한 기회를 찾을 수 있는 환경을 만든다.

```
[현재 — 기록 없는 경력]

일함 → 기억에 의존 → 이직 시 기억 소실 → 빈 이력서 → 미스매치 채용 → 조기 퇴사
                                                         ↓
                                              사회적 비용 증가
                                              (재채용 비용, 생산성 손실)

[CareerLog 이후 — 기록된 경력]

일함 → 자동 기록·축적 → 이직 시 데이터 기반 이력서 → 정확한 매칭 → 적합한 채용
                                                         ↓
                                              사회적 비용 감소
                                              (채용 효율 향상, 직업 만족도 증가)
```

                    (양측 적합도 사전 확인)
                            ↓
                    높은 매칭 정확도
                            ↓
                    채용 만족도 향상, 조기 퇴사 감소
```

### 5. 데이터 윤리 및 개인정보 보호 방침

CareerLog는 사용자의 **가장 민감한 개인 정보 중 하나인 경력 데이터**를 다루는 서비스다. 따라서 데이터 윤리와 개인정보 보호를 서비스 설계의 최우선 원칙으로 둔다.

#### 기본 원칙

| 원칙 | 내용 |
|------|------|
| **① 데이터 주권은 사용자에게** | 모든 경력 데이터의 소유권은 전적으로 사용자에게 있다. CareerLog는 데이터를 보관·처리하는 수탁자이며, 소유자가 아니다. |
| **② 명시적 동의 없이 활용 없음** | 사용자의 명시적 동의(opt-in) 없이 경력 데이터를 제3자에게 공유하거나, 마케팅·분석 목적으로 활용하지 않는다. |
| **③ 최소 수집 원칙** | 서비스 제공에 필수적인 데이터만 수집한다. 불필요한 개인 식별 정보(주민등록번호, 상세 주소 등)는 수집하지 않는다. |
| **④ 언제든 삭제 가능** | 사용자는 언제든 자신의 모든 데이터를 완전히 삭제할 수 있다. 삭제 요청 후 30일 이내에 백업 포함 전량 파기한다. |
| **⑤ 투명한 운영** | 데이터 처리 방식, AI 분석 과정, 제3자 공유 여부를 개인정보처리방침에 명확히 공개하고, 변경 시 사전 고지한다. |

| **산업 이용 가능성** | HR Tech·경력 관리·채용 플랫폼 등에 직접 적용 가능 |

### 2. 기존 솔루션 대비 기술 차별점

기획서의 경쟁사 분석을 기술 관점에서 확장하여, CareerLog의 **기술적 포지셔닝**을 명확히 한다.

#### 기술 차별점 비교 (확장)

| 비교 항목 | Resume.io / Rezi | 사람인 / 잡코리아 | LinkedIn | 노션 / 옵시디언 | **CareerLog** |
|-----------|------------------|------------------|----------|----------------|---------------|
| **데이터 입력 방식** | 사용자가 폼에 직접 입력 | 사용자가 폼에 직접 입력 | 사용자가 직접 작성 | 사용자가 자유 형식으로 작성 | **대화형 자연어 + 파일 업로드 → AI 자동 파싱** |
| **AI 활용 범위** | 문장 다듬기, 키워드 삽입 | 이력서 포맷팅, 추천 공고 | 문구 제안 (LinkedIn AI) | 없음 | **분류·구조화·수치 감지·이력서 생성·JD 매칭·페르소나 심사까지 전 과정** |
| **구조화 엔진** | 없음 (사용자 입력 그대로) | 없음 | 없음 | 없음 | **직무별 STAR 프레임 자동 매핑 + 수치 성과 자동 감지** |
| **데이터 축적** | 1회성 (작성 후 끝) | 지원 이력만 저장 | 프로필 수동 업데이트 | 사용자 의지에 전적으로 의존 | **일상 기록이 시계열로 자동 누적 → 경력 자산화** |
| **시맨틱 검색** | 없음 | 키워드 검색만 | 키워드 검색만 | 전문 검색 (텍스트 매칭) | **벡터 임베딩 기반 자연어 시맨틱 검색** |
| **멀티 에이전트 아키텍처** | 없음 | 없음 | 없음 | 없음 | **career-recorder → resume-writer → job-analyzer 3단계 파이프라인** |
| **파일 분석** | 없음 | 없음 | 없음 | 없음 (수동 정리) | **PDF·PPT·Figma·엑셀 자동 파싱 → 경력 통합** |
| **출력 개인화 수준** | 템플릿 기반 범용 출력 | 정형 포맷 | 정형 프로필 | 없음 | **축적된 실제 업무 데이터 기반 고도 개인화 출력** |
| **JD 매칭** | 키워드 매칭 수준 | 키워드 기반 추천 | 키워드 기반 추천 | 없음 | **경력 데이터 vs JD 구조 분석 → 다차원 적합도 산출** |
| **데이터 이동성** | 제한적 | 제한적 | 제한적 | 자유 (로컬 파일) | **JSON·CSV 내보내기, 데이터 주권 보장** |

                       높음
                        │
          노션(수동)     │     ★ CareerLog
          옵시디언       │     (AI 자동 구조화 +
                        │      시계열 축적)
                        │
 [데이터 축적     ──────┼────── [데이터 축적
  없음/1회성]           │        지속적 누적]

### 3. 모방 장벽 — 후발 주자가 쉽게 따라올 수 없는 이유

CareerLog의 모방 장벽은 **단일 기술이 아닌, 시간이 지날수록 강화되는 복합 요소**로 구성된다.

#### 모방 장벽 구조

```
[모방 장벽 레이어]

Layer 1: 기술적 장벽 (초기)
├── 멀티 에이전트 파이프라인 설계 노하우
├── 직무별 프롬프트 엔지니어링 최적화
└── 파일 파서 다양성 (PDF·PPT·Figma·엑셀)

Layer 2: 데이터 장벽 (시간 경과 시 강화)
├── 사용자별 경력 데이터 축적량
├── 직무별 구조화 패턴 학습 데이터
└── 수정 피드백 기반 프롬프트 고도화 데이터

Layer 3: 네트워크/록인 장벽 (성숙기)
├── 사용 기간에 비례하는 전환 비용
├── B2B 고객의 HR 시스템 연동 비용
└── 헤드헌팅사 제휴 네트워크
```

#### 각 장벽 레이어 상세

**Layer 1: 기술적 장벽**

| 장벽 요소 | 모방 난이도 | 설명 |
|-----------|-------------|------|
| **멀티 에이전트 파이프라인** | 🟡 중간 | career-recorder → resume-writer → job-analyzer의 3단계 파이프라인은 단순 LLM API 래핑이 아님. 에이전트 간 데이터 흐름·상태 관리·에러 핸들링 등 실제 서비스 운영 노하우가 필요하며, 이는 Pre-MVP 단계부터 축적해온 **실전 경험**에서 비롯됨 |
| **직무별 프롬프트 엔지니어링** | 🔴 높음 | PM, 디자이너, 개발자, 마케터 등 직무별로 업무 기록의 패턴·용어·성과 표현 방식이 다름. 각 직무에 최적화된 프롬프트·Few-shot 예시를 확보하는 데는 **수백~수천 건의 실제 사용 데이터 기반 튜닝**이 필요 |
| **파일 파싱 다양성** | 🟡 중간 | PDF·PPT·Figma·엑셀 등 다양한 포맷의 업무 산출물을 파싱하여 경력으로 통합하는 기술은 포맷별 엣지 케이스 처리 경험이 누적되어야 안정화됨 |
| **STAR 프레임 자동 매핑 정확도** | 🔴 높음 | 비정형 텍스트에서 Situation·Task·Action·Result를 정확히 분리·매핑하는 것은 프롬프트만으로 해결되지 않으며, 직무별·맥락별 최적화와 사용자 피드백 반영이 지속적으로 필요 |

**Layer 2: 데이터 장벽 (시간이 지날수록 강화)**

| 장벽 요소 | 강화 메커니즘 | 후발 주자의 불리함 |
|-----------|-------------|-------------------|
| **사용자별 경력 데이터 축적** | 사용자가 6개월, 1년, 3년 기록을 쌓을수록 CareerLog 내 경력 자산의 밀도가 증가 → 다른 서비스로 이전 시 이 데이터를 처음부터 다시 쌓아야 함 | 후발 서비스는 **"빈 상태에서 시작"** → CareerLog 대비 출력 품질이 구조적으로 낮을 수밖에 없음 |
| **직무별 구조화 패턴 데이터** | 수천~수만 명의 사용자 기록을 처리하면서 직무별 업무 표현 패턴·수치 성과 표현 패턴이 데이터로 축적됨 | 후발 서비스는 이 패턴 데이터가 없으므로 **구조화 정확도에서 열위** |
| **사용자 수정 피드백 데이터** | AI 구조화 결과를 사용자가 수정할 때마다, 오류 패턴이 로깅되어 프롬프트 개선에 활용됨 (기술 개발 계획의 "AI 출력 품질 관리 루프" 참조) | 이 피드백 루프가 **6개월 이상 돌아야** 프롬프트 품질이 안정화됨 — 후발 주자는 이 학습 기간을 단축할 수 없음 |

```
[데이터 장벽의 선순환 구조]

사용자 기록 입력
      │
      ▼
AI 구조화 ──→ 사용자 확인 ──→ 수정 발생 시 피드백 로깅
      │                              │
      │                              ▼
      │                     프롬프트 개선 (주간)
      │                              │
      ▼                              ▼

      │

      │
      ▼

```

> 이 선순환은 서비스 출시 후 **시간이 지날수록 가속**되며, 후발 주자가 동일한 수준에 도달하려면 최소 **12~18개월의 데이터 축적 기간**이 필요하다 (추정). 이것이 CareerLog의 가장 강력한 모방 장벽이다.

**Layer 3: 네트워크/록인 장벽 (성숙기)**

| 장벽 요소 | 전환 비용 | 시기 |
|-----------|-----------|------|
| **개인 사용자 록인** | 1년 이상 기록을 축적한 사용자가 다른 서비스로 이전할 경우, 축적된 경력 데이터의 맥락(프로젝트 연결, 시계열 흐름, AI가 학습한 개인 패턴)을 그대로 가져갈 수 없음. 데이터 내보내기(JSON/CSV)는 제공하나, **맥락까지의 이전은 불가** | 2년차 이후 본격화 |
| **B2B 고객 록인** | 기업이 CareerLog Team 플랜을 도입하고 HR 시스템(SAP SuccessFactors, Workday 등)과 연동한 이후에는 **시스템 교체 비용**이 매우 높아짐 | 3년차 이후 |
| **헤드헌팅사 제휴 네트워크** | CareerLog 유저 풀 기반으로 구축된 헤드헌팅사 제휴 관계는 후발 서비스가 동일한 규모의 유저 풀 없이 복제할 수 없음 | 2~3년차 |

#### 오픈소스 기여 전략 (선택적)

| 항목 | 방침 | 근거 |
|------|------|------|
| **경력 데이터 표준 포맷** | 2~3년차에 오픈 스펙 공개 검토 | 경력 데이터의 표준 포맷(JSON Schema)을 공개하면, 업계 표준으로 자리잡아 CareerLog의 플랫폼 지위를 강화할 수 있음. 데이터 이동성 보장은 사용자 신뢰 확보에도 기여 |
| **범용 유틸리티 라이브러리** | 파일 파서 등 범용 도구의 일부를 오픈소스로 공개 검토 | 개발자 커뮤니티 내 인지도 확보 + 채용 브랜딩 효과. 단, 핵심 로직(프롬프트·파이프라인)은 공개하지 않음 |
