## 한 줄 정의

**일하는 동안 PC가 알아서 경력을 쌓는다. 이직할 때 꺼내 쓰면 된다.**

> **포지셔닝 (2026.03.20 PD 검토 후 확정)**
> "귀찮음 자동화"가 아니라 **"업무 블랙박스"** — 이직할 때, 연봉 협상할 때, 성과 평가 때 내가 한 일의 증거가 없어서 손해 본 경험을 해소한다.

## 1. 문제 정의

### 대부분의 직장인은 자기 경력을 모른다

- 퇴직 직전에 이력서를 쓰려 하면 "3년 동안 내가 뭘 했지?"가 된다
- 업무는 매일 하지만 기록이 없다
- 이직 결심 후 2주 안에 경력을 정리하려다 포기한다
- 결국 직함과 회사명만 쓴 "빈 이력서"로 지원한다
- **면접관 앞에서, 연봉 테이블 앞에서, 성과 평가 앞에서 — 증거가 없어서 손해 본다**

### 기존 솔루션의 한계

| 도구 | 문제 |
|------|------|
| 노션/옵시디언 | 기록은 되지만 구조화·자동화 없음. 결국 안 씀 |
| 링크드인 | 본인이 직접 써야 함. 업데이트 귀찮아서 방치 |
| AI 이력서 툴 | 작성 도움만. 기억 자체가 없으면 소용없음 |
| 타임트래킹 툴 (토글 등) | 시간만 기록. "어떤 성과였는지" 해석 불가 |
| 일반 메모앱 | 경력 맥락 없이 흩어짐 |

### 핵심 인사이트

> 문제는 "이력서 작성 능력"이 아니라 **"기록되지 않은 경험"** 이다.
> 그리고 기록이 안 되는 이유는 단순하다 — **기록하려면 행동을 멈춰야 하기 때문이다.**

## 2. 솔루션

### CareerLog — PC가 알아서 기록하는 업무 블랙박스

CareerLog는 **Electron 데스크톱 앱**이다. 사용자가 아무것도 입력하지 않아도 된다.

"업무 시작" 버튼 하나로 PC가 블랙박스로 변한다. 어떤 앱을 얼마나 썼는지, 어떤 프로젝트에 시간을 쏟았는지 — 10초마다 자동으로 쌓인다. 업무가 끝나면 AI가 그 기록을 STAR 포맷의 경력 기술서로 바꿔준다.

```
업무 시작 버튼 클릭
    ↓
PC 포그라운드 창 자동 감지 (10초 간격, PowerShell)
    ↓
타임라인 자동 생성 (앱별/시간순, 유휴 시간 감지 포함)
    ↓
업무 종료 → 프로젝트 태깅 (선택)
    ↓
AI가 STAR 포맷 경력 기록 자동 생성
    ↓
경력 대시보드 누적 → 이직·협상·평가 시 꺼내 씀
```

**사용자가 하는 일은 단 두 가지다 — 시작 버튼, 그리고 이직할 때 복사 붙여넣기.**

## 3. 핵심 기능

### Phase 1 — 자동 기록 (✅ v0.2 구현 완료)

| 기능 | 설명 |
|------|------|
| **PC 활동 자동 감지** | 포그라운드 창을 10초마다 폴링. 어떤 앱을 얼마나 썼는지 무입력 자동 기록 |
| **Idle 감지** | 15분 이상 자리 비움 시 자동 감지. 해당 시간은 기록에서 조용히 제외 (다이얼로그 없음) |
| **세션 시작/종료** | "업무 시작/종료" 버튼으로 세션 단위 기록. 트레이 상주로 백그라운드 추적 |
| **프로젝트 태깅** | 업무 종료 후 이 세션이 어떤 프로젝트였는지 태깅 (건너뛰기 옵션 포함) |
| **수치 자동 감지** | AI가 활동 내용에서 성과 수치 ("오류 75% 감소" 등)를 자동 하이라이트 |

### Phase 2 — 조회·관리 (✅ v0.2 구현 완료)

| 기능 | 설명 |
|------|------|
| **타임라인 뷰** | 세션별 활동 기록 시각화. 앱별 그룹 뷰 / 시간순 토글 |
| **커리어 대시보드** | 히어로 카드(이번 주 현황), 누적 카드(총 업무 시간·기록 수), 히트맵, 앱 TOP3, 경력기록 미리보기 |
| **주간 알림** | 주 1회 "이번 주 기록됨" 시스템 트레이 알림으로 리텐션 확보 |
| **날짜 필터** | 타임라인 날짜 피커로 과거 세션 조회 |

## 4. 타겟 유저

### 메인 타겟

**직장인 3~7년차, 이직 예비자**

- 경력은 있는데 정리가 안 된 사람
- 지금 당장 이직 생각은 없지만 "나중에 필요하겠다"는 걸 아는 사람
- 포트폴리오가 없는 기획자·PM·디자이너·개발자
- 연봉 협상·성과 평가 때 근거 없이 밀린 경험이 있는 사람

> 타겟의 핵심 심리: "지금 쓰는 게 아니라 쌓아두는 것"
> CareerLog는 이직 도구가 아니라 **경력 보험**이다.

### 세컨더리 타겟

- **신입 취준생** — 인턴·프로젝트 경험 체계적 기록
- **프리랜서·N잡러** — 다수 클라이언트 프로젝트 관리 및 포트폴리오 축적

> **B2B(HR 담당자)는 Phase 5 이후로 연기.** B2C 기록 가치 증명이 선행되어야 B2B 신뢰 확보 가능.

### 페르소나 예시

| 이름 | 상황 | CareerLog 사용 동기 |
|------|------|-------------------|
| 백지민 (30, Product Designer) | 다이버스에서 3년, 이직 준비 중. 한 게 많은데 정리가 안 됨 | "지난 3년치 Figma 작업시간·산출물 증거가 필요하다" |
| 변지현 (32, 게임기획) | QA·기획·PM 멀티롤. 뭘 어필해야 할지 모름 | "내가 어떤 프로젝트에 얼마나 기여했는지 숫자로 보고 싶다" |
| 이민준 (28, 개발자) | 사이드 프로젝트 많은데 이력서에 어떻게 쓸지 모름 | "오늘 뭘 얼마나 코딩했는지 자동으로 기록됐으면" |

## 5. 수익 모델

### 전략 원칙

> B2C 구독을 먼저 증명한다. "기록 자체를 유료화"하되, 끊으면 기록 접근이 위협받는 구조로 지속 구독 동기를 만든다. B2B는 B2C 트랙션 확보 후 진입한다.

### B2C 구독

| 플랜 | 가격 | 내용 |
|------|------|------|
| **Free** | 무료 | 세션 20개까지, AI 경력 기록 생성 월 3회, 내보내기 제한 |
| **Pro** | 월 9,900원 | 무제한 세션·기록, AI 생성 무제한, 전체 포맷 내보내기, JD 매칭 |
| **Premium** | 월 19,900원 | Pro + 현직자 페르소나 심사, 모의 면접, 이직 전략 리포트 |

> **Free 제한 설계 원칙:** 기록은 계속 쌓이게 하되, 꺼내 쓸 때 (내보내기·AI 생성) 유료 전환. 기록을 쌓은 사람일수록 이탈 비용이 높아진다.

### B2B (Phase 5 이후)

| 플랜 | 가격 | 내용 |
|------|------|------|
| **Team** | 월 인당 5,000원 | 직원 성과 기록, 팀 대시보드, 관리자 뷰 |
| **Enterprise** | 별도 협의 | HR 시스템 연동, SSO, 온보딩 지원 |

> B2B 진입 조건: B2C MAU 5,000 이상 또는 유료 전환 300명 이상 달성 후 협상 테이블 구성.

### 파트너십 수익 (장기)

- 잡코리아·원티드 등 채용 플랫폼 제휴 → 경력 데이터 기반 채용 연결 레퍼럴
- 이직 성공 시 헤드헌팅사 레퍼럴 커미션

## 6. 경쟁사 분석

| 서비스 | 강점 | CareerLog와의 차이 |
|--------|------|-------------------|
| Resume.io / Rezi | AI 이력서 작성 보조 | 작성 도움만. 기억 자체를 저장하지 않음 |
| 사람인 / 잡코리아 | 국내 채용 플랫폼 지배력 | 기록·축적 기능 없음. 이직 결심 후에만 사용 |
| 링크드인 | 글로벌 커리어 네트워크 | 본인이 직접 써야 함. 업데이트 방치 패턴 |
| 노션 / 옵시디언 | 자유로운 지식 관리 | 경력 특화 구조 없음. AI 없음. 결국 안 씀 |
| 토글 / 클락카이파이 | 시간 추적 | 시간만 기록. "어떤 성과였는지" 해석 불가 |
| **CareerLog** | **무입력 자동 기록 + AI 경력 해석** | **유일한 포지션** |

### 핵심 차별화

> 경쟁사들은 모두 **"이미 기억하고 있는 것"을 잘 쓰게 도와준다.**
> CareerLog는 **"기억할 필요 없게"** 만든다.
> 그리고 블랙박스처럼 쌓인 기록은 증거가 된다 — 연봉 협상 테이블에서, 성과 평가 회의실에서.

## 7. 시장 규모 (TAM/SAM/SOM)

## 8. 기술 스택

### 현재 구현 (v0.2 기준)

| 영역 | 기술 | 비고 |
|------|------|------|
| 앱 프레임워크 | Electron + Vite + React | electron-forge, Squirrel 인스톨러 |
| 활동 감지 | PowerShell 영구 프로세스 (spawn 1회, 내부 루프) | Win32 GetForegroundWindow API, Add-Type 1회 컴파일 후 10초 루프, active-win 대체 |
| 스토리지 | JSON 파일 (로컬) | sessions / activities / career_records / app_settings |
| AI | Claude API (Sonnet) | STAR 포맷 경력 기록 생성, 직군·직급·스킬 프롬프트 주입 |
| 로컬 AI 서버 | 순수 Node.js http 모듈, port 3759 | express 미사용 (패키징 이슈) |
| 빌드·배포 | electron-forge make → Squirrel .exe | Windows x64 |

### 기술 전략 원칙

- **로컬 우선:** 모든 데이터는 사용자 PC에 저장. 서버 없음. 프라이버시 소구점.
- **신뢰가 제품의 생사:** 잘못 감지된 기록 하나가 전체 신뢰를 무너뜨린다. 정확도 개선이 기능 추가보다 우선.
- **Mac 확장:** Windows PowerShell 의존 구조를 플랫폼 추상화 레이어로 교체하는 것이 Mac 지원의 핵심 기술 과제.

### 향후 추가 예정

| 영역 | 기술 | 시점 |
|------|------|------|
| Mac 지원 | osascript 기반 창 감지, dmg 빌드 | Mac 환경 확보 후 |
| 클라우드 동기화 | Railway 등 별도 백엔드 | B2B 전환 시 |

## 10. 엑시트 전략

### 1순위: 국내 채용 플랫폼 인수

잡코리아·원티드·사람인 계열은 "채용 공고"는 있지만 "구직자의 경력 원장"이 없다. CareerLog의 축적 데이터는 이들이 보유하지 못한 자산이다.

- **파트너십 → 인수 루트:** JD 매칭·추천 기능 연동으로 트래픽·데이터 가치 증명 후 인수 협상
- **협상 레버리지:** 유료 구독자 수 + 누적 기록 세션 수 + D30 리텐션

### 2순위: HR SaaS 전략적 인수

성과 관리·OKR SaaS (레몬베이스 등) 계열의 "개인 경력 기록" 기능 확장 수단으로 전략적 인수 가능성.

### 3순위: 독립 성장

국내 채용 플랫폼 종속을 피하고 글로벌 (일본·동남아) 확장으로 독립 SaaS 포지셔닝.

> **PD 코멘트:** B2B 영업과 M&A 딜을 동시에 진행하기 어렵다. 파트너십 먼저, 인수 협상은 나중. 링크드인(MS)은 한국 시장 규모만으로 현실성 낮음.

- B2C 구독: 불안 기반 락인이 아니라 "기록 자체의 가치"로 구독 동기 설계.

- B2B 영업과 M&A 딜 동시 진행 불가 — 파트너십 먼저, 인수 협상 나중.

- 결핍 지표 절대 금지 (목표 달성률, 미작성 기록 수 등)
