## 1. 문제 정의

### 대부분의 직장인은 자기 경력을 모른다

- 퇴직 직전에 이력서를 쓰려 하면 "3년 동안 내가 뭘 했지?"가 된다
- 업무는 매일 하지만 기록이 없다
- 이직 결심 후 2주 안에 경력을 정리하려다 포기한다
- 결국 직함과 회사명만 쓴 "빈 이력서"로 지원한다

### 기존 솔루션의 한계

| 도구 | 문제 |
|------|------|
| 노션/옵시디언 | 기록은 되지만 구조화·자동화 없음. 결국 안 씀 |
| 링크드인 | 본인이 직접 써야 함. 업데이트 귀찮아서 방치 |
| AI 이력서 툴 | 작성 도움만. 기억 자체가 없으면 소용없음 |
| 일반 메모앱 | 경력 맥락 없이 흩어짐 |

### 핵심 인사이트

> 문제는 "이력서 작성 능력"이 아니라 **"기록되지 않은 경험"** 이다.

---

## 2. 솔루션

### CareerLog — 일하면서 자동으로 쌓이는 경력 저장소

대화로 오늘 한 일을 말하면, AI가 알아서 구조화해서 저장한다.
쌓인 기록은 언제든 이력서·경력기술서·LinkedIn으로 꺼낼 수 있다.

```
오늘 업무 입력 (대화/음성/파일)
        ↓
AI가 분류·구조화·저장
        ↓
프로젝트별 / 직무별 / 연도별 경력 누적
        ↓
이직 시 → 이력서·경력기술서 자동 생성
         → 지원 전략·피드백 제공
```

## 3. 핵심 기능

### Phase 1 — 기록 (MVP)

| 기능 | 설명 |
|------|------|
| **대화형 업무 기록** | "오늘 이런 거 했어"를 말하면 AI가 구조화해서 저장 |
| **파일 분석 저장** | PDF·PPT·Figma 등 업무 산출물을 분석해서 경력에 통합 |
| **자동 분류** | 프로젝트명·직무 카테고리·날짜 자동 태깅 |
| **수치 감지** | "오류 75% 감소" 같은 수치 자동 하이라이트 및 성과로 저장 |

### Phase 2 — 조회·관리

| 기능 | 설명 |
|------|------|
| **타임라인 뷰** | 연도별·회사별·프로젝트별 경력 시각화 |
| **성과 대시보드** | 저장된 수치 성과 모아보기 |
| **검색** | "내가 Figma 관련해서 한 게 뭐가 있지?" 자연어 검색 |

### Phase 3 — 출력

| 기능 | 설명 |
|------|------|
| **이력서 자동 생성** | 쌓인 경력 기반 이력서 초안 자동 생성 |
| **경력기술서 자동 생성** | 문제→접근→결과 스토리 구조로 자동 작성 |
| **LinkedIn 최적화** | 섹션별 최적화 텍스트 생성 |
| **JD 매칭** | 채용공고 붙여넣으면 내 경력과 매칭 분석 |

### Phase 4 — 피드백·전략

| 기능 | 설명 |
|------|------|
| **현직자 페르소나 심사** | 목표 회사 직군 팀장 인격으로 서류 심사·피드백 |
| **모의 면접** | 저장된 경력 기반 예상 질문 + 답변 코칭 |
| **이직 전략 리포트** | 현재 경력 갭 분석 + 보완 로드맵 제안 |

## 4. 타겟 유저

### 메인 타겟

**직장인 3~7년차 이직 준비자**
- 경력은 있는데 정리가 안 된 사람
- 이직 결심했지만 이력서 쓰기 막막한 사람
- 포트폴리오가 없는 기획자·PM·디자이너

### 세컨더리 타겟

- **신입 취준생** — 인턴·프로젝트·수업 경험 기록
- **프리랜서** — 다수 클라이언트 프로젝트 관리
- **HR 담당자 (B2B)** — 직원 성과 기록 + 인사 연동

### 페르소나 예시

| 이름 | 상황 |
|------|------|
| 백지민 (30, Product Designer) | 다이버스에서 3년, 이직 준비 중. 한 게 많은데 정리가 안 됨 |
| 변지현 (32, 게임기획) | QA·기획·PM 멀티롤. 뭘 어필해야 할지 모름 |
| 이민준 (28, 개발자) | 사이드 프로젝트 많은데 이력서에 어떻게 쓸지 모름 |

## 6. 경쟁사 분석

| 서비스 | 강점 | 우리와의 차이 |
|--------|------|--------------|
| Resume.io / Rezi | AI 이력서 작성 | 작성 도움만, 기억 저장 없음 |
| 사람인 / 잡코리아 | 국내 채용 플랫폼 강점 | 기록·축적 기능 없음 |
| LinkedIn | 글로벌 네트워크 | 본인이 직접 써야 함 |
| 노션 | 자유로운 기록 | 경력 특화 구조·AI 없음 |
| **CareerLog** | **일상 기록 → 경력 자동 축적** | **유일한 포지션** |

### 핵심 차별화

> 경쟁사들은 모두 **"이미 기억하고 있는 것"을 잘 쓰게 도와준다.**
> CareerLog는 **"기억할 필요 없게"** 만든다.

## 7. 기술 스택 (MVP 기준)

### 이미 검증된 것 (Harry 프로젝트 기반)

- Claude API — 대화형 기록, 분석, 생성
- 에이전트 파이프라인 — career-recorder → resume-writer → job-analyzer
- 파일 분석 — PDF·Figma·엑셀 파싱 후 경력 통합

| 백엔드 | Node.js / FastAPI |
