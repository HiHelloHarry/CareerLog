# CareerLog 경력 기록 뷰 리디자인 기획서

**작성:** 이수진 (UX)
**작성일:** 2026-03-25
**버전:** v1.0

---

## 리디자인 철학

CareerLog의 핵심 가치는 "버튼 두 개로 매일 커리어가 자동으로 쌓이는 것"이다. 경력 기록 뷰는 **매일 쌓인 기록을 한눈에 조망하는 커리어 대시보드**가 기본 화면이며, STAR 상세는 이력서 작성 등 필요할 때만 꺼내 보는 기능이다. 평소에는 카드 요약과 캘린더로 "이만큼 쌓였다"를 확인하고, 이력서가 필요한 순간에만 드로어를 열어 STAR/불릿을 활용한다.

## 0. 문제 정의

### 현재 상태
| 항목 | 현재 | 문제 |
|------|------|------|
| 사이드바 | 148px, 날짜별 1:1 플랫 리스트 | 300개 기록 시 스크롤 지옥. 시간 기반 탐색 불가 |
| 메인 영역 | 선택 즉시 STAR 전문 노출 | 일상적 조회에 과도한 정보량. "어떤 기록이 있었지?" 파악이 힘듦 |
| 네비게이션 | 프로젝트 필터만 존재 | 시간축 탐색 없음. 분기/월 단위 회고 불가 |
| 활용 시점 | 항상 STAR 전체가 기본값 | 평소엔 훑어보기, 이력서 작성 시에만 STAR 필요 |

### 사용자 시나리오 분석

**시나리오 A — 일상 점검 (주 2-3회, 30초)**
> "이번 주에 뭐 했더라?" 빠르게 훑고 싶다. STAR 전문은 필요 없다.

**시나리오 B — 월간/분기 회고 (월 1회, 5분)**
> "이번 분기에 어떤 프로젝트를 했지?" 기간별로 묶어서 보고 싶다.

**시나리오 C — 이력서 작성 (반기 1회, 30분+)**
> 특정 기록의 STAR를 열어서 복사하거나 편집해야 한다. 여러 기록을 비교해야 한다.

**시나리오 D — 연봉 협상/성과 정리 (연 1-2회, 1시간)**
> 특정 기간의 기록을 한꺼번에 내보내거나, 핵심 성과만 추려야 한다.

## 1. 정보 계층 설계

### Layer 구조

```
L0  시간 네비게이션 (년/분기/월)
    "어느 시점의 기록을 볼 것인가"

L1  기록 카드 리스트 (요약 뷰)
    "그 시점에 어떤 기록들이 있는가" — 한 줄 요약 + 프로젝트 + 스킬 칩

L2  기록 상세 (STAR 뷰)
    "특정 기록을 깊이 보겠다" — 패널 확장 또는 드로어
```

### 설계 원칙
1. **L0와 L1이 기본 화면이다.** STAR(L2)는 명시적 액션으로만 진입한다.
2. **시간축이 최상위 네비게이션이다.** 프로젝트 필터는 시간축 하위에 놓는다.
3. **한 화면에서 최대한 많은 기록을 조망한다.** 카드는 높이를 최소화한다.
4. **액션은 맥락 가까이.** 복사/내보내기/수정은 L2에서만 노출한다.

## 3. 컴포넌트 상세 명세

### 3-1. L0: 시간 네비게이션 바

**위치:** 메인 영역 상단 우측
**형태:** 가로 스크롤 가능한 분기 피커

```
구조:  [년도 라벨]  < [Q4] [Q1] [Q2] >
       2025        ← prev  active  next →
```

| 속성 | 값 |
|------|-----|
| 기본 선택 | 현재 분기 |
| 클릭 동작 | 해당 분기의 기록만 필터링 + 월별 섹션 구분 |
| "전체" 옵션 | 분기 피커 왼쪽에 "전체" 탭 → 모든 기록 (월별 그룹핑 유지) |
| 빈 분기 | 비활성 스타일 (클릭 불가) |
| 화살표 | 이전/다음 분기 이동. 데이터 없는 범위 넘어가면 숨김 |

**분기 계산 로직:**
- Q1: 1-3월, Q2: 4-6월, Q3: 7-9월, Q4: 10-12월
- records의 created_at에서 min/max 추출하여 존재하는 분기만 표시

### 3-2. 서브 필터 바

**위치:** 분기 피커 아래, 카드 그리드 위

```
[프로젝트: 전체 v]  [키워드 검색 ...]          23개 기록 · 127시간
```

| 요소 | 설명 |
|------|------|
| 프로젝트 드롭다운 | 기존 프로젝트 필터 유지. 선택된 분기 내 프로젝트만 옵션에 표시 |
| 검색 | 기록 content/star 텍스트 내 키워드 검색. 디바운스 300ms |
| 통계 라벨 | "N개 기록" + 해당 분기의 총 추적 시간 (sessions에서 계산) |

### 3-3. L1: 기록 카드 (RecordCard)

**레이아웃:** 2열 그리드, 월별 구분선
**카드 높이:** 고정 높이 없음, 내용에 따라 가변이되 max-height 120px + overflow hidden

```
+----------------------------------+
| 3.24 (월) · CareerLog 개발       |   <- 날짜 + 프로젝트명

| 통계 API 연동 완료                |      (최대 2줄, ellipsis)
| [React] [Electron] [Chart]       |   <- skills 칩 (최대 3개 + "+N")
| ● 수치 포함                      |   <- metrics_detected 있을 때만
+----------------------------------+
```

| 속성 | 값 |
|------|-----|
| 배경 | var(--bg2) |
| 테두리 | 1px solid var(--border) |
| 모서리 | border-radius: 12px |
| 호버 | border-color: var(--a), 미세한 translateY(-1px) |
| 활성(선택됨) | border: 1.5px solid var(--a), background: var(--a-dim) |

| 프로젝트명 | fontSize 11, color var(--ink3), 날짜 오른쪽에 " · " 구분 |

| 스킬 칩 | 기존 chip chip-b 스타일, fontSize 10.5 |
| 수치 배지 | fontSize 10.5, color var(--g), fontWeight 700 |
| 수정됨 표시 | 카드 우상단에 작은 도트 (var(--a), 6px circle) |

**요약 텍스트 추출 우선순위:**
1. `star.bullets[0]` (불릿 첫 번째 항목)
2. `star.situation` 첫 문장

| 액션 바 | 드로어 하단 고정 (sticky bottom) |

### CareerResult.jsx (리팩토링 후 구조)

```jsx
// 의사 코드 — 실제 구현 시 참고용

function CareerResult({ content, star, records, ... }) {
  const [quarterFilter, setQuarterFilter] = useState(() => getCurrentQuarter())
  const [projectFilter, setProjectFilter] = useState('')
  const [searchQuery, setSearchQuery] = useState('')
  const [selectedId, setSelectedId] = useState(null)

  const availableQuarters = useMemo(() => extractQuarters(records), [records])
  const filteredRecords = useMemo(() => applyFilters(...), [records, quarterFilter, projectFilter, searchQuery])
  const groupedByMonth = useMemo(() => groupByMonth(filteredRecords), [filteredRecords])

  // 방금 생성된 기록이 있으면 자동으로 드로어 열기
  useEffect(() => {
    if (content && !selectedId) {
      const newRecord = records?.find(r => /* 가장 최근 */)
      if (newRecord) setSelectedId(newRecord.id)
    }
  }, [content])

  return (
    <div className="career-result-container">
      {/* 헤더 + L0 */}
      <div className="career-header">
        <h2>경력 기록</h2>
        <QuarterNav
          quarters={availableQuarters}
          selected={quarterFilter}
          onChange={setQuarterFilter}
        />
      </div>

      {/* 필터 바 */}
      <FilterBar
        projects={projects}
        projectFilter={projectFilter}
        onProjectChange={setProjectFilter}
        searchQuery={searchQuery}
        onSearchChange={setSearchQuery}
        recordCount={filteredRecords.length}
      />

      {/* L1 카드 그리드 */}
      <RecordGrid
        groupedByMonth={groupedByMonth}
        selectedId={selectedId}
        onSelect={setSelectedId}
        newContent={content}
      />

      {/* L2 드로어 */}
      {selectedId && (
        <DetailDrawer
          record={filteredRecords.find(r => r.id === selectedId)}
          records={filteredRecords}
          onClose={() => setSelectedId(null)}
          onNavigate={setSelectedId}
          onDelete={handleDelete}
          onRegenerate={handleRegenerate}
          onSave={handleSaveEdit}
          onCopy={handleCopy}
          onExport={handleExport}
        />
      )}
    </div>
  )
}
```

## 7. 사용자 감정 흐름 설계

| 단계 | 사용자 행동 | 기대 감정 | 설계 의도 |
|------|------------|-----------|-----------|
| 진입 | 경력 기록 탭 클릭 | "한눈에 보인다" | 카드 그리드가 즉시 로드. 분기별 정리된 느낌 |
| 탐색 | 분기 탭 전환 | "정리가 잘 되어있네" | 월별 구분선이 시간 흐름을 안내 |
| 발견 | 카드 훑어보기 | "이런 것도 했었지" | 요약 텍스트 + 스킬 칩이 기억을 환기 |
| 진입 | 카드 클릭 | "더 자세히 볼까" | 드로어가 부드럽게 슬라이드. 기존 뷰는 유지 |
| 활용 | 복사/내보내기 | "바로 쓸 수 있겠다" | 드로어 하단에 액션이 항상 보임 |
| 복귀 | 드로어 닫기 | "다른 것도 볼까" | 리스트 위치 유지. 컨텍스트 손실 없음 |
