## 1. 제품 개요

### 1-1. 한 줄 정의

> **일하는 순간마다 경력이 자동으로 쌓이는 데스크탑 앱 — 이직할 때 꺼내 쓰면 된다.**

### 1-2. 제품이 해결하는 문제

대부분의 직장인은 퇴직 직전에야 이력서를 쓰려 한다. 그 순간 3~5년간 자신이 무슨 일을 했는지 구체적으로 기억하지 못한다. 노션·옵시디언 같은 도구로 기록을 시도하더라도 구조 없이 흩어지거나, 귀찮아서 결국 멈춘다.

핵심 문제는 "이력서 작성 능력"이 아니라 **기록되지 않은 경험** 자체다.

| 기존 솔루션 | 한계 |
|---|---|
| 노션 / 옵시디언 | 수동 기록, 구조화 없음, 지속성 낮음 |
| LinkedIn | 직접 써야 함, 업데이트 방치 |
| AI 이력서 툴 | 기억이 없으면 도움 불가 |
| 일반 메모앱 | 경력 맥락 없이 흩어짐 |

### 1-3. 솔루션 개요

CareerLog는 백그라운드에서 사용자가 어떤 앱·문서를 얼마나 썼는지를 30초 단위로 자동 추적한다. 업무 종료 시 타임라인을 보여주고, AI(Claude)가 활동 내역을 이력서용 경력 기술 문장으로 변환한다. 사용자는 기억력에 의존하지 않고 매일의 업무를 체계적으로 경력으로 쌓을 수 있다.

```
[시스템 트레이 상주]
       ↓
업무 시작 → 30초마다 활성 앱/창 자동 감지
       ↓
업무 종료 → 타임라인 리뷰 + 메모 추가
       ↓
Claude AI → STAR 구조 경력 기록 생성
       ↓
경력 저장소 누적 → 이력서/LinkedIn/자기소개서 출력
```

### 1-4. 타겟 유저

**Primary:** 직장인 3~7년차 이직 준비자
- 경력은 있는데 정리가 안 된 기획자·PM·디자이너·개발자
- 이직 결심 후 이력서 쓰기가 막막한 사람
- 수치가 있는 업무 성과를 기억 못 하는 사람

**Secondary:**
- 신입 취준생 — 인턴·프로젝트 경험 정리
- 프리랜서 — 다수 클라이언트 프로젝트 관리
- 부업/사이드 프로젝트 병행자

### 1-5. 플랫폼 및 기술 스택

| 영역 | 기술 |
|---|---|
| 앱 프레임워크 | Electron (Windows / macOS) |
| UI | React + Tailwind CSS |
| 활성 창 감지 | `active-win` (npm) |
| 로컬 저장 | JSON 파일 (sessions, activities, career records) |
| AI 분석 | Claude API (claude-opus-4-6) |
| 빌드/배포 | Vite + Electron Forge |

## 2. 현재 버전 기능 정의 (v0.1)

v0.1에서 구현 완료된 기능을 개발자가 유지보수하거나 v0.2를 이어서 작업할 때 기준이 될 수 있도록 상세히 기술한다.

### 2-1. 시스템 트레이 앱

**동작 방식**
- Electron 앱이 시작되면 시스템 트레이에 아이콘이 등록된다.
- 트레이 우클릭 메뉴: `▶ 업무 시작` / `⏹ 업무 종료` / `창 열기` / `종료`
- 트레이 더블클릭 시 메인 창이 열린다.
- 기록 중일 때 트레이 툴팁이 `CareerLog — 기록 중`으로 변경되고, 가장 최근 활성 앱명이 메뉴에 표시된다.
- `app.on('window-all-closed')` 이벤트를 막아 창을 닫아도 앱이 종료되지 않도록 처리됨.

**파일 위치:** `electron-app/src/main.js`

### 2-2. 홈 화면 (Home)

**비기록 상태 (isTracking = false)**
- API 키 미설정 시 상단에 앰버색 배너 표시 → 클릭 시 설정 탭으로 이동
- 프로젝트/클라이언트명 입력 필드 (선택, Enter 키로 시작 가능)
- `업무 시작` 버튼 — 클릭 시 `window.careerlog.startTracking(project)` IPC 호출
- 확인 오버레이 출력: "업무 기록이 시작되었습니다" + 트레이 사용법 안내
- `확인, 창 닫기` 버튼 → `window.close()`

**기록 중 상태 (isTracking = true)**
- 경과 시간 타이머 (HH:MM:SS, 초 단위 카운트업)
- 파란색 맥동 애니메이션 "기록 중" 표시
- `업무 종료` 버튼 → `window.careerlog.stopTracking()` → 타임라인 화면으로 전환

**파일 위치:** `electron-app/src/components/Home.jsx`

### 2-3. 활동 추적 (Tracker)

**동작 방식**
- `startTracking()` 호출 시 `setInterval`로 30초마다 `active-win` 실행
- 반환값: `{ owner: { name }, title }` → app_name, window_title로 저장
- 마지막 활동과 앱명이 같으면 체류 시간 연장, 다르면 새 활동 레코드 생성
- `stopTracking()` 호출 시 마지막 활동의 ended_at을 현재 시각으로 flush

**저장 구조 (로컬 JSON)**
```
sessions.json     — 세션 목록 (id, date, started_at, ended_at, project)
activities.json   — 활동 목록 (id, session_id, app_name, window_title,
                               started_at, ended_at, duration_sec, memo)
```

**파일 위치:** `electron-app/src/tracker.js`, `electron-app/src/database.js`

### 2-4. 타임라인 화면 (Timeline)

**화면 구성**
- 세션 요약 헤더 카드 (날짜, 총 기록 시간, 활동 수, 프로젝트명 태그)
- 경력 기록 생성 버튼 (API 키 있으면 활성, 없으면 "API 키 설정하기"로 대체)
- 타임라인 세로 선 + 각 활동 아이템

**ActivityItem 구성 요소**
- 앱 아이콘 (앱명 기반 이모지 매핑 — Figma, Slack, Zoom, Chrome, VSCode 등)
- 앱명 + 체류 시간 칩
- 창 제목 (truncate 처리)
- 시작/종료 시각 (HH:MM 포맷)
- 병합 표시: `🔗 N개 통합` 칩 (merged_ids.length > 1인 경우)
- 메모 입력: 없으면 `+ 메모 추가`, 있으면 앰버색 칩으로 표시 → 클릭 시 인라인 편집

**활동 병합 로직**
- 체류 시간 5분(300초) 미만이고 같은 앱명의 연속 활동은 하나로 병합
- 병합된 아이템은 `opacity-55`로 약하게 표시 (단독 short 활동은 흐리게)
- 병합된 경우 앰버색 테두리 + `🔗 N개 통합` 칩으로 시각적 구분

**파일 위치:** `electron-app/src/components/Timeline.jsx`

### 2-5. AI 경력 기록 생성

**동작 방식**
1. 타임라인 화면의 `경력 기록 생성` 버튼 클릭
2. API 키 유무 재확인 → 없으면 설정 화면으로 리다이렉트
3. `window.careerlog.generateCareerRecord(sessionId)` IPC 호출
4. Main 프로세스에서 세션의 activities 조회 → Claude API 전송
5. 프롬프트: 활동 목록(앱명, 창 제목, 체류 시간, 메모) 전달 → 이력서용 경력 기술 문장 생성 요청
6. 응답 content를 career_records에 저장 후 화면 전환

**사용 모델:** `claude-opus-4-6`

**응답 형식:** 자유 텍스트 (마크다운 bullet 형식)

**파일 위치:** `electron-app/src/main.js` (IPC 핸들러), `electron-app/src/ai.js`

### 2-6. 경력 기록 화면 (CareerResult)

**구성**
- 왼쪽 사이드바: 과거 기록 목록 (날짜/시각, `수정됨` 표시)
- 메인 영역: 경력 기록 내용 (pre 태그로 마크다운 pre-wrap 렌더링)
- 헤더 액션 버튼: `수정` / `복사` (수정 모드에서는 `저장` / `취소` / `복사`)
- 수정 모드: textarea로 전환, `window.careerlog.updateCareerRecord(id, content)` 호출

**파일 위치:** `electron-app/src/components/CareerResult.jsx`

### 2-7. 설정 화면 (Settings)

**구성 항목**
- Anthropic API 키 입력/저장 (비밀번호 타입 input, 마스킹 표시)
- API 키 필요 이유 설명 박스
- Anthropic Console 링크 (외부 브라우저 오픈)
- 감지 설정 표시 전용 (30초 감지 간격, 5분 병합 기준, 버전 v0.1.0)
- 모든 기록 삭제 버튼 (2단계 확인: 첫 클릭 → 4초 내 재클릭으로 확정)

**파일 위치:** `electron-app/src/components/Settings.jsx`

### 2-8. 네비게이션 구조

## 3. 발견된 기능 Gap 및 우선순위 분석

careerly-v2 프로토타입 분석 및 제품 비전과 현재 v0.1 구현 사이의 간격을 정리한다.

### 3-1. Gap 목록 및 우선순위

| # | 기능 | 현황 | 우선순위 | 선정 이유 |
|---|---|---|---|---|
| G-01 | **구조화된 STAR 경력 기록** | 자유 텍스트 bullet | **P0** | 핵심 가치 제안. 현재 결과물의 품질이 사용 지속성을 결정 |

| G-04 | **연속 기록 스트릭 (Streak)** | 없음 | **P0** | 매일 사용 습관 형성. 리텐션에 직접 영향 |

| G-09 | **태그 / 스킬 자동 추출** | 없음 | **P1** | AI가 경력 기록 시 사용 기술 태그(React, Figma 등) 추출 → Dashboard 통계 기반 |
| G-10 | **Dashboard / 통계** | 없음 | **P2** | 총 경력 수, 스킬 분포, 프로젝트 수 시각화. 동기 부여에 기여하나 초기에는 부차적 |
| G-11 | **Search — 경력 검색** | 없음 | **P2** | 기록이 쌓인 후 필요. 초기 유저에게는 아직 불필요 |
| G-12 | **여러 산출물 생성** | 이력서 bullet만 | **P2** | LinkedIn / 자기소개서 / 포트폴리오 출력. 핵심이나 기록 품질 먼저 해결 후 확장 |
| G-13 | **Profile / 사용자 정보** | 없음 | **P2** | 이름, 직함, 프로필 이미지. UI 완성도를 높이나 기능적 우선순위는 낮음 |

| G-15 | **Import 기능** | 없음 | **P3** | Notion/Google Docs에서 이관. 성숙 단계에서 필요 |

### 3-2. P0 선정 근거

P0는 v0.2에서 **반드시** 해결해야 하는 항목이다. 기준은 다음 두 가지다:

1. **핵심 가치 제안의 품질**에 직접 영향을 주는가 → G-01(STAR), G-02(수치 감지)
2. **사용 루프 완성 및 리텐션**에 직접 영향을 주는가 → G-03(Done 화면), G-04(Streak)

## 4. v0.2 상세 기획

### 4-1. 연속 기록 스트릭 (Streak) — P0

#### 기능 설명

사용자가 며칠 연속으로 경력 기록을 저장했는지를 시각화한다. 매일 CareerLog를 사용하는 습관을 형성하고 리텐션을 높이는 핵심 메커니즘이다.

"오늘도 기록했다"는 즉각적 보상을 제공하며, 스트릭이 끊기지 않도록 하는 동기를 부여한다.

#### 사용자 시나리오

1. 사용자가 오늘 업무를 마치고 경력 기록을 저장한다.
2. Done 화면 (4-5 참조)에서 "3일 연속 기록 중!" 메시지와 스트릭 바가 표시된다.
3. 다음 날 홈 화면을 열면 상단에 스트릭 바 (7일 도트)가 보인다.
4. 오늘 기록을 완료한 날은 도트가 골드색으로 채워진다.
5. 기록하지 않은 날은 빈 원, 오늘은 테두리 강조로 구분된다.

#### UI/UX 상세

**홈 화면 스트릭 바 컴포넌트**

위치: 홈 화면 로고 아래, 날짜 위

```
최근 7일
  월  화  수  목  금  토  일
  ●   ●   ●   ○   ●   ○  [오늘]
             (미기록) (주말) (테두리)
```

- 도트 크기: 10px × 10px, 원형
- 채워진 도트: `#d4a84b` (골드)
- 빈 도트: `#2a2a30` (다크 서피스, 테두리 `#3a3a42`)
- 오늘 도트: 테두리 두께 2px, `#d4a84b` 색상의 ring

- 스트릭 카운트 텍스트: `N일 연속 기록 중` — 도트 바 오른쪽 정렬, 12px, 골드색

### 4-2. 수동 입력 모드 — Chat UX — P1

자동 추적 결과에 보충 입력할 수도 있고, 완전히 수동으로만 입력하는 것도 가능하다.

#### 사용자 시나리오

1. 홈 화면에서 `직접 입력하기` 버튼을 탭한다.
2. Chat UX 화면이 열린다. AI가 "오늘 어떤 업무를 하셨나요?" 로 시작한다.
3. 사용자가 "오늘 신규 온보딩 플로우 기획하고 팀 리뷰 미팅 했어요"라고 입력한다.

5. 사용자가 답변 후 "완료" 또는 퀵 리플라이 칩 중 하나를 선택.

### 4-3. 구조화된 STAR 경력 기록 — P0

#### 기능 설명

현재 AI 출력은 자유 텍스트 bullet 형태다. v0.2에서는 Claude가 다음 5개 필드를 가진 구조화된 JSON을 반환하도록 프롬프트를 개선하고, UI에서 각 필드를 별도 카드로 렌더링한다.

| 필드 | 설명 | 예시 |
|---|---|---|
| **Situation** | 배경/상황 | "신규 사용자 온보딩 전환율이 31%로 업계 평균 대비 낮은 상황" |
| **Task** | 맡은 역할/목표 | "온보딩 플로우 전면 재설계 리드 (PM 1인, 디자이너 1인)" |
| **Action** | 구체적 행동 | "사용자 인터뷰 6건, 경쟁사 벤치마킹 → 3단계 → 2단계로 단순화" |
| **Result** | 결과/성과 | "온보딩 전환율 31% → 58% 달성 (3개월 내)" |
| **Skills** | 사용 기술/스킬 | `["UX Research", "Figma", "A/B Testing", "Jira"]` |

7. Skills 태그 리스트 표시 + 태그 추가/삭제 가능
8. 수치 발견 시 Result 카드에 `수치 발견!` 골드 뱃지 (G-02와 연계)

### 4-4. 수치 자동 감지 & 하이라이트 — P0

#### 기능 설명

창 제목, 메모, AI 생성 텍스트에서 숫자가 포함된 성과 지표를 자동으로 감지하여 강조 표시한다. 이력서에서 수치 성과는 채용 담당자의 주목도를 크게 높이기 때문에, 사용자가 수치를 놓치지 않도록 능동적으로 알린다.

**감지 대상 패턴**

| 패턴 유형 | 예시 |
|---|---|
| 퍼센트 | `전환율 31% → 58%`, `오류율 75% 감소` |
| 절대 수치 + 단위 | `MAU 12만`, `DAU 3,200`, `사용자 1,000명` |
| 금액 | `매출 2억`, `비용 30% 절감` |
| 시간 단위 성과 | `3주 만에`, `배포 주기 2주 → 3일` |
| 배율 | `2배 성장`, `3x 개선` |

#### 사용자 시나리오

1. 사용자가 창 제목 `온보딩_v3_전환율58프로.fig`로 작업

4. 수치 텍스트 `58%`, `27%p`가 골드색으로 하이라이트
5. 타임라인 화면 해당 활동 카드에도 `📊 수치 포함` 마이크로 칩 표시

```javascript
  /\d+[만억천백]+/g,                    // 한국어 수량 단위
  /\d{1,3}(,\d{3})+/g,                  // 쉼표 구분 숫자
  /\d+[xX배](\s|$)/g,                   // 배율
  /\d+\s*(명|건|회|개|일|주|달|년)/g,  // 단위 포함 수치
];

function detectMetrics(text) {
  const found = [];
  METRIC_PATTERNS.forEach(pattern => {
    const matches = text.match(pattern);
    if (matches) found.push(...matches);
  });
  return [...new Set(found)];
}
```

AI 프롬프트에서도 `metrics_detected` 필드로 추출을 요청하므로 두 레이어에서 감지됨.

#### 엣지 케이스

| 상황 | 처리 방식 |
|---|---|
| 수치 없는 세션 | 수치 관련 UI 요소 미표시 |
| 날짜/시각이 수치로 감지될 경우 | 패턴 필터 강화 — `YYYY-MM-DD`, `HH:MM` 패턴 제외 |
| 무의미한 수치 (파일 버전 `v2`, `_3`) | 단독 숫자가 아닌 경우 (앞뒤 알파벳 포함) 제외 처리 |

### 4-7. 앱 제외 목록 (Blacklist) — P1

#### 기능 설명

사용자가 경력 추적에서 제외하고 싶은 앱을 지정할 수 있는 기능이다. YouTube, Netflix, 게임, 음악 앱 등 업무와 무관한 앱이 타임라인에 섞이면 AI 분석 품질이 저하되고, 사용자에게 불편함을 준다.

#### 사용자 시나리오

1. 설정 화면에서 `앱 제외 목록` 섹션 확인
2. 현재 추적된 앱 목록 드롭다운에서 제외할 앱 선택
3. 또는 텍스트로 직접 앱명 입력
4. `추가` 버튼 클릭 → 제외 목록에 등록
5. 이후 추적 시 해당 앱이 활동 데이터에 저장되지 않음
6. 기존에 저장된 해당 앱 활동은 타임라인에서 `[제외됨]` 처리 (삭제는 아님)

#### UI/UX 상세

### 4-8. Activity 수동 편집 — P1

#### 기능 설명

자동 추적된 활동 목록에서 불필요한 항목을 삭제하거나, 잘못 감지된 항목의 앱명/제목을 수정할 수 있는 기능이다. 현재는 메모만 추가 가능하며, 활동 자체를 편집하거나 삭제할 수 없어 AI 분석 품질이 저하될 수 있다.

#### 사용자 시나리오

1. 타임라인 화면에서 특정 활동 카드를 롱-호버(또는 더보기 버튼 클릭)

5. `앱명 수정` → 인라인 input으로 앱명 변경 가능 (잘못된 앱 감지 교정)

#### UI/UX 상세

**활동 카드 편집 UI**

`⋯` 클릭 시 드롭다운 메뉴:
- `✏️ 앱명 수정` — 앱명을 인라인 input으로 변경
- `✂️ 분할` — 시간 기준으로 두 개 활동으로 분리 (v0.3으로 연기)

### 6-3. 우선순위 선정 근거

**P0 우선 이유**

CareerLog의 핵심 사용자 가치는 "매일의 업무가 고품질 경력 기록으로 자동 전환된다"는 것이다. 현재 v0.1은 이 가치 제안의 절반만 구현돼 있다. 자유 텍스트로 생성된 경력 기록은 이력서 즉시 활용이 어렵고, 완료 피드백이 없어 행동 강화가 이루어지지 않는다.

- **STAR 구조** 없이는 출력 결과물이 이력서에 바로 쓸 수 없는 품질이다.
- **수치 감지** 없이는 사용자가 가장 중요한 성과 수치를 놓치게 된다.
- **Done 화면** 없이는 "기록됐다"는 인지가 불분명해 사용 루프가 불완전하다.
- **Streak** 없이는 매일 사용 습관 형성이 어렵고 Day 7 리텐션이 낮아진다.

**P1 우선 이유**

P1 항목들은 사용 품질을 높이고 사용 시나리오를 확장한다:

- **Blacklist**: 데이터 오염 방지 — AI 분석에 YouTube, 게임 앱이 포함되면 결과 품질 저하
- **Activity 편집**: 삭제 없이는 타임라인이 "정리 안 된 기록"이 되어 사용자 신뢰 하락
- **Calendar**: 기록 히스토리가 시각화되지 않으면 "쌓이는 경력" 가치를 체감 못함
- **Chat UX**: 회의·전화 등 추적 불가 상황을 커버하지 못하면 기록이 불완전해짐
- **스킬 태그**: Dashboard와 검색의 토대가 되므로 P2 기능 전에 준비 필요

**P2 → P3 연기 이유**

- Dashboard/통계는 기록이 30개 이상 쌓인 후 의미 있음 (신규 사용자에게 불필요)
- 검색도 마찬가지로 기록 누적 후 필요 기능
- 클라우드 동기화는 유료화 단계에서 결제 모델과 함께 설계 필요
- Import는 기존 사용자 마이그레이션 용도로 초기 단계 불필요
