## 1. 문제 정의 (Problem)

### 핵심 문제: 직장인은 일하면서 경력을 쌓지 못한다

국내 사무·전문직 직장인의 경력 기록 실패는 의지의 문제가 아니다. 구조적 문제다.

**이직 결심 후 경력 정리를 포기하는 이유는 단 하나다: "기록하려면 일을 멈춰야 한다."**

- 직장인 3~7년차는 이직 결심 후 평균 2주 안에 경력 정리를 시도하다 포기한다
- "어떤 프로젝트를 했지?" → 기억이 없다 → 포기 → 이직 기회 소멸
- 문제는 경력이 없는 것이 아니라, 경력이 기록되지 않는 것이다

**기존 솔루션의 공통 맹점:** 노션, 링크드인, AI 이력서 툴, 타임트래킹 서비스는 모두 "이미 기억하는 것"을 잘 쓰게 도와줄 뿐이다. 기억 자체를 쌓아주는 서비스는 존재하지 않는다.

| 기존 솔루션 | 방식 | 한계 |
|---|---|---|
| 노션 / 메모 앱 | 직접 입력 | 기록하려면 행동을 멈춰야 함 |
| 링크드인 | 직접 작성 | 과거 회고 방식, 기억 의존 |
| 타임트래킹 (Toggle 등) | 수동 타이머 | 시작/종료 직접 입력 필요 |
| AI 이력서 툴 | 입력 기반 생성 | 입력할 내용이 없으면 무용 |
| CareerLog | **PC가 알아서 감지** | — |

## 2. 솔루션 (Solution)

### "업무 블랙박스" — 일하는 동안 PC가 알아서 경력을 쌓는다

CareerLog는 Windows 데스크톱 앱이다. 사용자가 하는 일은 두 가지뿐이다.

1. **업무 시작 버튼을 누른다**
2. **이직할 때 복사해서 붙여넣는다**

그 사이에 일어나는 모든 것은 CareerLog가 자동으로 처리한다.

### 작동 방식

```
업무 시작 버튼
    ↓
PC 포그라운드 창 10초마다 자동 감지 (PowerShell + Win32 GetForegroundWindow API)
    ↓
앱별 활동 타임라인 자동 생성
    ↓
업무 종료 버튼
    ↓
Claude AI가 STAR 포맷 경력 기록 자동 생성
    ↓
Markdown / HTML 내보내기, 클립보드 복사
```

### 핵심 기능 (MVP 구현 완료, v0.6 기준)

| 기능 | 상태 |
|---|---|
| PC 활동 자동 감지 (10초 폴링, Idle 15분 감지) | 완료 |
| 세션 시작/종료, 프로젝트 태깅, 작업 유형 선택(8개 카테고리) | 완료 |
| 타임라인 뷰 (앱별 그룹 / 시간순 토글) | 완료 |
| 커리어 대시보드 (히트맵, 스트릭, 인사이트 카드) | 완료 |
| AI STAR 경력 기록 자동 생성 (Claude Sonnet API) | 완료 |
| 직군·직급·스킬 프로필 AI 프롬프트 주입 | 완료 |
| Markdown/HTML 내보내기, 클립보드 복사 | 완료 |
| Windows x64 Setup.exe 인스톨러 | 완료 |

### 기술 차별화: 완전 로컬, 프라이버시 우선

- 활동 데이터는 사용자 PC 로컬에만 저장 (외부 서버 전송 없음)
- PowerShell + Win32 API 직접 호출 방식으로 네이티브 모듈 의존성 제거
- AI 생성 시에만 Claude API 호출 (활동 원본 데이터 미전송)

### PD 검토 결과 (김도현, 前 네이버·카카오·라인·당근마켓)

- 포지셔닝 확정: **"업무 블랙박스"** — 시장에 유사 컨셉 없음
- 핵심 리스크로 "기록 정확도 신뢰" 지목 → Win32 API 직접 감지 방식으로 해소 (UIAutomationClient 창 제목 오염 버그 수정 완료)
- 리텐션 전략: **축적 가시화** — 연금 잔고처럼 쌓이는 감각 (히트맵, 스트릭 카운터로 구현)

| **SOM** | ARR 6~24억원 | 초기 18개월 내 유료 전환 5,000~20,000명 (베타 리텐션 데이터 확보 후 수정 예정) |

### 타겟 사용자

**메인 타겟: 직장인 3~7년차, 이직 예비자**
- 기획자, PM, 디자이너, 개발자 (PC 기반 전문직)
- 경력은 있으나 정리가 안 된 상태, 이직 동기 높음
- Windows PC 사용률 높고, 로컬 앱에 대한 신뢰도 높음

**세컨더리 타겟:**
- 신입 취준생 (포트폴리오 구축 수단)
- 프리랜서·N잡러 (프로젝트별 실적 정리 수요)

### 경쟁 구도

CareerLog의 포지셔닝은 "기록 자동화 × 이력서 출력 품질"을 동시에 해결하는 유일한 서비스다. 현존하는 어떤 경쟁사도 이 두 축을 동시에 충족하지 않는다.

```
                  이력서 출력 품질 高
                        │
         링크드인        │     CareerLog (목표)
         잡코리아AI      │
─────────────────────────┼─────────────────────────
 기록 수동          │        기록 자동
─────────────────────────┼─────────────────────────
         노션·메모       │     Toggle + AI 조합
                        │     (아직 존재 안 함)
                  이력서 출력 품질 低
```

**목표:** D30 리텐션 40% 달성, 핵심 버그 제로화

- Windows x64 Setup.exe 배포 (Squirrel 인스톨러)
- 지인 기반 베타 유저 10~30명 모집
- 사용 데이터 기반 UX 개선 (핵심: 기록 정확도 신뢰 확보)
- 주요 측정 지표: DAU, 세션 시작 횟수, AI 생성 전환율, D7/D30 리텐션

**기술 완성도:** MVP v0.6 현재 베타 배포 준비 완료 상태

### Phase 2 — B2C 구독 결제 도입 (3~4개월)

**목표:** MRR 500만원 달성

- 결제 시스템 연동 (Pro/Premium 플랜)
- AI 무제한 생성 서버 안정화 (Railway 배포)

- JD 매칭 기능 개발 (채용 공고 키워드 기반 경력 기록 최적화)

### Phase 3 — Mac 지원 + MAU 5,000 달성 (5~8개월)

**목표:** 크로스 플랫폼 확장, 유기적 성장 채널 확보

- macOS 버전 개발 (활동 감지 엔진 플랫폼별 분기)
- 콘텐츠 마케팅 (이직 커뮤니티, 커리어 관련 SNS 채널)
- 온보딩 최적화로 신규 유저 활성화율 제고

### Phase 4 — 잡코리아·원티드 파트너십

**목표:** 채용 플랫폼 API 연동, 유통 채널 확보

- JD 자동 매칭 + 지원서 1클릭 생성 기능
- 파트너십 협상 기반: 베타 사용자 데이터, 전환율 지표

### Phase 5 — B2B 진출 및 엑시트 (12개월 이후)

**목표:** B2B 팀 플랜 출시, 전략적 투자 유치 또는 인수 협상

- 기업 팀 플랜 (월 인당 5,000원)

  1. 잡코리아·원티드·사람인 계열 전략적 인수
  2. HR SaaS (레몬베이스 등) 통합
  3. 글로벌 독립 성장 (일본·동남아)

### 사회적 가치

**직장인의 경력 자본 격차 해소**

현재 이직 시장에서 경력 정리 품질은 개인의 글쓰기 능력과 시간 여유에 따라 크게 달라진다. CareerLog는 일하는 동안 자동으로 경력을 축적함으로써, 경력 정리에 투입할 시간이 없는 직장인도 동일한 출발선에 설 수 있게 한다.

- 이직 준비 기간 단축 → 노동 시장 유동성 증가
- AI 생성 경력 기록의 민주화 → 전문직 정보 비대칭 완화
- 로컬 저장 방식으로 직장인 개인정보 보호

### 투자 유치 및 엑시트 시나리오

국내 커리어테크 시장은 잡코리아(2021년 1조 7,900억원 카카오 인수), 원티드랩(상장 후 시총 최대 5,000억원) 등의 사례가 보여주듯 대형 M&A 수요가 확인된 시장이다. CareerLog는 채용 플랫폼이 아직 보유하지 못한 "업무 기록 자동화" 레이어를 선점하는 전략으로, 플랫폼 사업자의 전략적 인수 대상이 될 수 있다.
