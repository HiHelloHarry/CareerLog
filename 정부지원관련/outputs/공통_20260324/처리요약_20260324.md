## 처리 상세

### 예비창업패키지 (합격 76점)

**심사 점수:** 심사위원 A 73점 / B 79점 / C 76점

**반영한 피드백 항목:**

1. **[기술 신뢰도 보강]** UIAutomationClient 오염 버그 발견·수정 이력을 기술 차별점으로 명시. "잘 작동한다"는 주장에서 "버그 발견 경로와 해결 방식"으로 기술 신뢰 근거를 구체화. (심사위원 B: 자동 감지 정확도 수치 부재 지적에 대한 대응)

2. **[팀 리스크 대응 구체화]** 공동창업자 탐색을 "창업 커뮤니티 및 지인 연결망을 통해 GTM·그로스 역량 보유 후보 탐색 진행 중"으로 명시. 외주 개발자에 대한 기술 스펙 문서 사전 제공 및 주간 코드 리뷰 계획 추가. (심사위원 A·C: 단독 창업 리스크 지적에 대한 대응)

3. **[초기 100명 모집 전략 구체화]** "블라인드 이직 카테고리 후기 글 → 직군 커뮤니티 타겟 초대 → 지인 네트워크 보조" 순서의 단계별 실행 계획 명시. 비지인 채널 피드백을 의사결정 우선 기준으로 삼는다는 원칙 추가. (심사위원 C: "100명이 목표가 아닌 바람으로 들린다"는 지적에 대한 대응)

**유지한 항목 (기획서 미확인으로 지어낼 수 없는 것):**

- 고객 인터뷰 수치: `[창업자 확인 필요]` 표시로 삽입 위치 안내. 베타 전 수행 권장 사항으로 처리.

### TIPS (합격 73.3점)

**심사 점수:** 심사위원 A 72점 / B 71점 / C 77점

**반영한 피드백 항목:**

1. **[기술 증빙 안내]** 1인 개발 주장의 객관적 증빙으로 GitHub 공개 저장소 링크 또는 Setup.exe 다운로드 링크 삽입을 `[창업자 확인 필요]`로 명시. TIPS 연계 AC의 기술 실사(Technical Due Diligence) 대응을 위한 조치. (심사위원 A·B: 기술 증빙 전무 지적에 대한 대응)

2. **[공동창업자 탐색 구체화]** 채용 포지션별 목표 시점, 채용 채널, 구체적 역할을 표로 제시. "합류 논의 예정"에서 구체적 채용 계획으로 수준을 높임. (심사위원 C: "합류 논의 예정은 계획이 없는 것과 거의 같다" 지적에 대한 대응)

3. **[초기 100명 고객 획득 전략 구체화]** "블라인드 광고"에서 단계별 채널 전략(블라인드 후기 글 → 직군 커뮤니티 → 링크드인 콘텐츠 → 지인 보조)으로 구체화. (심사위원 C: "'블라인드에 광고 올리겠다'는 것은 전략이 아니다" 지적에 대한 대응)

**유지한 항목:**

- AI 품질 수치: 베타 전 단계로 측정값이 없으므로 지어내지 않음.
- 고객 인터뷰 데이터: `[창업자 확인 필요]` 없이 유지. 합격 확정 문서이므로 현재 사실 그대로 유지.

### Pre-TIPS (합격 72점)

**심사 점수:** 심사위원 A 69점 / B 73점 / C 74점

**반영한 피드백 항목:**

1. **[베타 유저 목표 상향]** Phase 1 베타 유저 목표를 "10~30명"에서 "최소 100명 (콜드 채널 포함)"으로 상향. 30명 이하 샘플에서는 리텐션 노이즈와 신호 구분이 불가하다는 심사위원 C 지적 반영.

2. **[창조경제혁신센터 계획 명시]** 지원 자격 요건과 직결되는 창조경제혁신센터 추천서 미확보 상황을 문서 상단에 명시하고, 접촉 단계별 계획을 `[창업자 확인 필요]` 형식으로 구체화. (심사위원 A·B·C: 지원 자격 요건 미해소 공통 지적에 대한 대응)

3. **[지재권 전략 방향 명시]** "[확인 필요]"만으로 처리되어 있던 특허·지재권 계획을 SW 저작권 등록, 특허 출원 검토, 영업비밀 지정 3가지 방향으로 구체화하되 `[창업자 확인 필요]`로 처리. (심사위원 B: 기술 모방 리스크 무방비 지적에 대한 대응)

**[확인 필요] 항목 유지 처리:**

- 창조경제혁신센터 추천 확보: 계획 단계이므로 확정 사실로 기재하지 않음. 접촉 단계별 계획으로 대체.
- 특허·지재권 확보: 변리사 상담 전이므로 확정 방향을 기재하지 않음.

### 초기창업패키지 (합격 75점)

**심사 점수:** 심사위원 A 76점 / B 71점 / C 80점

**반영한 피드백 항목:**

1. **[AI 생성 품질 증빙 안내]** AI 생성 결과물 샘플 1~2건 삽입을 `[창업자 확인 필요]`로 명시. "Figma 2시간 + Slack 30분 → STAR 경력 기록"의 실제 변환 품질이 이 제품의 핵심 가치이며, 2027년 1월 제출 전 반드시 포함해야 한다고 명시. (심사위원 B: STAR 생성 품질 검증 부재가 가장 큰 기술적 리스크라는 지적에 대한 대응)

2. **[팀 빌딩 구체화]** 채용 채널 명시 및 "공동창업자 후보 접촉 현황 업데이트" `[창업자 확인 필요]`로 명시. "지원금 후 채용" 계획이 실제로 진행 중인지 여부를 문서에 반영할 수 있도록 안내. (심사위원 A·B·C: 1인 체제 리스크 공통 지적에 대한 대응)

3. **[고객 목소리 보강 계획 구체화]** 문제 인터뷰 채널·대상·목표 건수·핵심 질문을 구체화하고 "2027년 1월 제출본에 반드시 포함" 안내. (심사위원 A·B·C 공통: 고객 목소리가 한 건도 없다는 지적에 대한 대응)

**제거한 항목:**

- 제출용에 부적합한 내부 작업 문구 전량 제거 (심사 기준 명시, 내부 메타 정보 등)
