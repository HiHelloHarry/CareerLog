## 1. 문제-솔루션 적합성 (PSF)

### 해결하는 문제: "경력 3년치가 기억에만 있다"

직장인이 이직·연봉 협상·성과 평가를 앞두고 공통으로 겪는 문제가 있다. 지난 수 년간 실제로 무엇을 했는지 구체적 수치로 설명할 기록이 없다는 것이다.

**직장인 34명 대상 설문 결과 (2026.03):**
- 경력·업무 이력을 **분기 이하 빈도로만 관리**: **88%**
- CareerLog 같은 자동 기록 도구 **사용 의향 있다**: **76%** (바로 써보고 싶다 29% + 관심 있다 47%)
- **유료 구독 의향 있다**: **71% (24명)**
- 가장 끌리는 기능 1위: **업무 활동 자동 기록 (47%)** — AI STAR 변환(44%)보다 "기록이 쌓이는 것" 자체에 대한 수요가 더 높음

- 이직 결심 후 경력 정리에 상당한 시간이 소요되며, 결국 직함과 회사명만 쓴 빈 이력서로 마무리된다.
- 연봉 협상·성과 평가 자리에서 수치 근거 없이 밀린 경험은 기획자·PM·디자이너 직군에서 반복된다.
- 문제는 이력서 작성 능력이 아니다. **기록되지 않은 경험**이 원인이다.

**기록이 생기지 않는 구조적 이유**

기록하려면 행동을 멈춰야 한다. 업무 중 별도로 기록하는 행위가 부담 — 즉, **귀찮기** 때문에 노션·옵시디언 같은 도구가 있어도 지속 사용되지 않는다. 설문에서도 이직할 때만 정리한다는 응답이 38%, 며칠에 걸쳐 정리한다는 응답이 44%로, 경력 관리의 구조적 비효율이 확인되었다.

**기존 솔루션이 채우지 못한 공백**

| 도구 | 한계 |
|------|------|
| 노션·옵시디언 | 기록 가능하나 구조화·자동화 없음. 지속 사용률 낮음 |
| 링크드인 | 수동 입력 필요. 업데이트 방치 패턴 고착 |
| AI 이력서 툴 (Resume.io, Rezi 등) | 작성 도움만 제공. 기억 자체가 없으면 소용없음 |
| 타임트래킹 툴 (Toggl 등) | 시간만 기록. "어떤 성과였는지" 해석 불가 |

**핵심 공백:** "귀찮음 제로로 매일 커리어가 자동으로 쌓이는" 카테고리 자체가 현재 시장에 없다.

### 솔루션: CareerLog — 버튼 두 개로 매일 커리어가 자동으로 쌓이는 업무 블랙박스

CareerLog는 Windows 데스크톱 앱이다. 사용자가 별도로 입력하지 않아도 된다. 귀찮음 제로로 매일 커리어가 쌓이는 유일한 자동 기록 서비스다.

"업무 시작" 버튼과 "업무 종료" 버튼 — 이 두 버튼만으로 매일 커리어가 자동으로 쌓인다. 어떤 앱을 얼마나 썼는지, 어떤 프로젝트에 시간을 쏟았는지 10초마다 자동으로 기록되고, 커리어 대시보드와 캘린더에 매일 축적된다. 필요할 때 AI가 쌓인 기록을 STAR 포맷 경력 기술서로 변환한다.

```
업무 시작 버튼
    ↓
PC 활동 자동 감지 (10초 간격, 앱별 파서 17개 + URL 추출 + 활동 강도 감지)
    ↓
타임라인 + 커리어 대시보드에 매일 커리어가 쌓임
    ↓
업무 종료 버튼
    ↓
필요할 때 → AI가 STAR 경력 기록 자동 생성 (작업 흐름 기반)
    ↓
경력 캘린더·카드에 누적 → 이직·협상·평가 시 꺼내 씀
```

사용자가 하는 일은 단 두 가지다 — 업무 시작, 업무 종료. STAR 경력 기록은 쌓인 기록에서 필요할 때 꺼내 쓰는 부산물이다.

**CareerLog가 이 문제에 적합한 이유**

기존 경쟁 솔루션은 모두 "이미 기억하고 있는 것"을 잘 쓰게 도와준다. CareerLog는 "기억할 필요 없게" 만든다. PSF가 성립하는 이유는 솔루션이 문제의 근본 원인(기록 행위의 부담 = 귀찮음)을 제거하기 때문이다. 설문에서 가장 끌리는 기능 1위가 "업무 활동 자동 기록(47%)"이었다는 점이 이를 뒷받침한다 — 사용자가 원하는 것은 AI 이력서가 아니라, 귀찮음 없이 매일 커리어가 쌓이는 것이다.

## 2. 기술 독창성

### "자동 기록(귀찮음 해소) × 커리어 대시보드(매일 확인) × AI STAR 변환(필요할 때)"

**혁신 1: OS-level 활동 자동 감지 + 정밀 파싱 (행동 개입 제로)**

기존 타임트래킹 서비스는 사용자가 타이머를 직접 시작·종료해야 한다. CareerLog는 Windows PowerShell + Win32 GetForegroundWindow() API를 직접 호출하여, 사용자 개입 없이 10초마다 포그라운드 창을 자동 감지한다.

- UIAutomationClient 방식의 창 제목 오염 버그를 Win32 API 직접 호출로 해결 — 기록 정확도 검증 완료
- 오픈소스 네이티브 모듈(active-win) 의존성 없이 구현하여 Electron 패키징 환경에서의 배포 안정성 확보
- 15분 Idle 자동 감지로 비활성 세션 오염 방지
- **(v0.7) 앱별 창 제목 파서 17개** — VS Code, Chrome, Edge, Slack, Figma, Notion, Excel, Word, PPT, Zoom, Teams, KakaoTalk, Postman, IntelliJ, WebStorm, Terminal 등 개별 앱의 창 제목을 구조화된 메타데이터(`parsed` 필드)로 변환
- **(v0.7) 브라우저 URL 추출** — UIAutomationClient로 Chrome/Edge 주소창 읽기, 민감 도메인은 호스트만·일반 도메인은 pathname depth 2까지만 저장 (프라이버시 보호)
- **(v0.7) 키보드·마우스 활동 감지** — `GetAsyncKeyState`(키보드) + `GetCursorPos`/`VK_LBUTTON`(마우스) → `inputLevel` = active/light/passive. 디자이너 등 마우스 중심 작업도 active로 판정
- **(v0.7) 작업 흐름(Flow) 감지** — 개별 앱 전환을 의미 단위로 묶음. 같은 프로젝트의 다른 파일은 하나의 flow, 5분 이하 이탈은 [보조] 태그, 이전 flow 재합류 허용

**혁신 2: 커리어 대시보드 — 매일 쌓이는 것이 보이는 경험 (v0.8)**

자동 기록된 데이터가 커리어 대시보드·캘린더·카드에 매일 축적된다. 사용자는 이직할 때만이 아니라 매일 자신의 커리어가 쌓이는 것을 확인할 수 있다.

- **(v0.8) 프로필 히어로** — 이번 주·총 업무일·경력기록 수 통계 + 주간 바차트 + TOP 사용 도구
- **(v0.8) 캘린더 뷰** — 월별 캘린더에 기록 있는 날짜마다 프로젝트 태그 표시, 클릭 시 해당 일자 카드 필터링
- **(v0.8) 업무 카드 2열 그리드** — 한줄 요약 + 스킬 칩으로 빠르게 스캔, STAR 전문은 카드 클릭 시 디테일 드로어에서만 노출
- **(v0.8) 디테일 드로어** — 오른쪽 슬라이드 패널에서 STAR/불릿 상세 확인, 키보드 좌우 이동, 복사·내보내기·수정·재생성 지원

**혁신 3: 맥락 인식 AI 경력 기록 생성 — 쌓인 기록에서 필요할 때 꺼내 쓰는 부산물**

PC 활동 데이터(앱 사용 패턴, 작업 유형, 프로젝트 태그, URL, inputLevel) + 작업 흐름(Flow) 분석 + 사용자 직군·직급·기술스택 프로필을 결합하여 Claude API가 STAR 포맷 경력 기록을 자동 생성한다.

- **(v0.7) Flow 기반 AI 프롬프트** — 개별 앱 사용 나열이 아닌, 의미 단위 작업 흐름으로 구조화된 입력. [핵심]/[보조] 역할 태그 + inputLevel 태그로 AI가 업무 맥락을 정확히 파악

출력 구조: `{situation, task, action, result, skills, metrics_detected, bullets}` — 이력서·포트폴리오·LinkedIn에 직접 활용 가능한 포맷

**AI 생성 결과물 품질 — 실제 사용 예시 (v0.7 Flow 기반):**

| Flow | 앱 (parsed) | 역할 | 체류 시간 | inputLevel | 메모 |
|---|---|---|---|---|---|
| Flow 1: 대시보드 UI 개편 | Figma (대시보드 v2 — 메인 화면.fig) | [핵심] | 2시간 12분 | active | 대시보드 레이아웃 리뷰 후 피드백 반영 |
| Flow 1 | Slack (디자인팀 채널) | [보조] | 32분 | active | — |
| Flow 1 | Notion (PRD 2.0, url: notion.so/PRD-v2) | [보조] | 45분 | light | — |
| Flow 1 | Jira (DASH-142~148, url: jira.com/browse/DASH) | [보조] | 1시간 16분 | active | — |

> **AI 출력 (STAR):** "대시보드 v2 UI 개편 프로젝트에서 Figma 기반 디자인 리뷰 주도, Jira 티켓 7건 반영 사항 확정. Slack·Notion 활용 디자인팀-기획팀 간 실시간 피드백 루프 운영, 핸드오프 4시간 45분 내 완료."

v0.7의 Flow 감지가 개별 앱 전환을 "대시보드 UI 개편"이라는 의미 단위로 자동 묶고, [핵심]/[보조] 역할 태그 + inputLevel로 AI가 업무 맥락을 정확히 파악한다. 메모 없이도 구조화된 기록만으로 맥락 복원이 가능하다.

**세 혁신의 결합이 만드는 진입장벽**

```
"기록 자동화" 단독 → 타임트래킹 서비스와 차별화 불충분
"AI 이력서 생성" 단독 → 입력 데이터가 없으면 무용
"커리어 대시보드" 단독 → 수동 입력이면 유지 안 됨
CareerLog = 자동 기록(귀찮음 해소) × 대시보드(매일 확인) × AI STAR(필요할 때) → 시장에 없는 조합
```

**프라이버시 설계**

모든 데이터는 사용자 PC 로컬에만 저장된다. 외부 서버 전송 없음. 이는 기술 선택(서버리스 로컬 JSON 아키텍처)의 결과이며, AI 도구 사용에 따른 개인정보 유출 우려를 구조적으로 차단한다.

**기술 보호 전략**

1. **SW 저작권 등록** — CareerLog 소스 코드를 한국저작권위원회에 프로그램 등록 (베타 출시 시점 수행)
2. **영업비밀 관리** — AI STAR 프롬프트 엔지니어링 로직과 활동 병합 알고리즘을 영업비밀로 지정, 접근 통제 체계 적용
3. **방어적 공개** — Win32 API 활용 패턴을 기술 블로그에 공개하여 선행기술화, 경쟁자의 특허 선점 차단

**AI API 의존성 관리:** 프롬프트 계층을 API-agnostic으로 설계하여 Claude, GPT, Gemini 등 주요 LLM 간 전환이 가능한 구조. Phase 2에서 오픈소스 LLM(Llama 등) 로컬 추론 옵션을 추가하여 API 비용 리스크를 헤징할 계획이다.
