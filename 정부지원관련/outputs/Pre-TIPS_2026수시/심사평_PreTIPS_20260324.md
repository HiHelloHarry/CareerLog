## 핵심 피드백 요약

### 강점
1. **메시징 리프레이밍 성공**: "AI STAR 자동 생성"에서 "버튼 두 개로 매일 커리어가 자동으로 쌓임"으로 전환. 설문에서 가장 끌리는 기능 1위가 "자동 기록(47%)"이라는 데이터가 이를 뒷받침하며, STAR는 "필요할 때 꺼내 쓰는 부산물"로 재배치한 것은 제품 본질에 더 부합.
2. **기술 완성도 v0.8**: Flow 감지 + inputLevel + 앱별 파서 17개 + 캘린더/대시보드 리디자인까지 3세대 고도화를 1인이 완수. Pre-TIPS 단계에서 이 수준의 MVP는 상위권. GitHub 공개 저장소로 기술 증빙이 가능한 점도 차별화.
3. **단위 경제 명확성**: Pro 4,900원 기준 마진율 69~93%, BEP 유료 1,000명, AI 비용 유저당 월 320원 등 수치 근거가 구체적. 가격 인하의 전략적 근거(설문 WTP 중심값)도 논리적.

### 보완 필수 항목

1. **[PSF 검증] 실사용 데이터 완전 부재**: 설문 n=34는 "의향"일 뿐 "행동"이 아님. Pre-TIPS 심사에서 가장 중요한 PSF 검증은 "실제로 설치하고 반복 사용한 사람이 있는가"임. --> 지인 10~20명에게 v0.8을 실제 설치시키고, 최소 D7 리텐션 + 정성적 피드백(재사용 의향, 불편사항)을 확보할 것. AI 생성 결과물 실제 캡처도 시뮬레이션이 아닌 실데이터로 교체 필수.

2. **[팀 역량] 1인 팀 리스크 미해소**: Pre-TIPS는 TIPS 연계를 전제하며, TIPS 운영사(VC)는 1인 팀에 투자하지 않음. "탐색적 대화 진행 중"은 구속력 없음. --> (최소) 기술 또는 GTM 역할의 공동창업자 후보 1명과 LOI(Letter of Intent) 수준의 합의를 확보하거나, 베타 기간 중 합류 확정 일정을 구체화할 것. 실제 이름/경력을 기재하지 못하더라도 "어떤 역량의 사람을 어떤 조건으로 영입할 것인지"가 더 구체적이어야 함.

3. **[사업화] "보험형 제품 vs 매일 행동 요구" 모순 미해소**: "경력 보험"으로 포지셔닝하면서 매일 "시작/종료" 버튼을 누르도록 요구하는 것은 사용자 행동 모델에서 충돌. 이 충돌이 해소되지 않으면 D30 리텐션 40%는 매우 도전적. --> (a) 백그라운드 자동 시작/종료 옵션(PC 켜면 자동 시작, 잠금 시 자동 종료) 또는 (b) 주 1회 요약 알림으로 "쌓인 것 확인"만 유도하는 패시브 모델을 Phase 1에서 테스트할 것을 권장. 사용자가 아무것도 안 해도 기록이 쌓이는 진정한 "블랙박스"가 되면 리텐션 문제가 구조적으로 해소됨.

### 추가 개선 권장 사항

4. **창조경제혁신센터 추천서**: Pre-TIPS 지원의 필수 자격 요건. 현재 "미완" 상태로, 이것이 확보되지 않으면 나머지 모든 점수가 무의미. 즉시 착수 필요.

5. **심사 제출본에서 액션 아이템 테이블 제거**: 문서 상단의 "미완" 항목 리스트는 내부 관리용. 심사위원에게는 "준비가 안 된 상태에서 제출했다"는 인상을 줄 수 있으므로 별도 문서로 분리할 것.

6. **Windows-only 제약과 타깃 고객의 불일치 해소**: 1순위 타깃인 기획자/디자이너 중 Mac 사용 비중이 높음. Phase 1 베타에서 Windows 사용자만 모집하는 것이 현실적이라면, 타깃 세그먼트를 "Windows 기반 사무직(개발자, PM, 회계/경영 직군)"으로 좁히고, Mac 확장은 로드맵에 명확히 배치할 것.
