## 핵심 피드백 요약

### 강점
1. **MVP 완성도가 높다** — v0.8까지 3세대 고도화, Win32 API + Flow 감지 + 17개 앱 파서 등 단순 PoC를 넘어선 실사용 수준의 기술 구현. 1인 개발로 이 수준을 만든 실행력은 Pre-TIPS 지원자 중 상위.
2. **고객 검증 시도가 있다** — 설문 34명으로 pain point 확인(분기 이하 관리 88%), 사용 의향(76%), 유료 의향(71%), WTP 기반 가격 설정까지 연결. 대부분의 Pre-TIPS 지원서가 고객 데이터 0건인 것을 감안하면 차별화 요소.
3. **메시징 리프레이밍이 효과적** — "AI 이력서 생성 도구"에서 "귀찮음 제로, 매일 커리어가 자동으로 쌓이는 서비스"로 전환. 이직 시점 1회성 도구가 아닌 일상적 습관 도구로 포지셔닝한 것이 구독 모델과 정합성이 높아짐.

### 보완 필수 항목 (합격선 안정적 확보를 위한 치명적 약점)

1. **[팀 역량] 1인 체제의 구체적 해소 계획 부재** -- "탐색적 대화 진행 중"은 확정이 아님. 심사위원 입장에서 1인 창업자는 Pre-TIPS까지는 수용할 수 있으나, TIPS 연계를 논하려면 최소 2인 이상이어야 함. --> **권장 수정 방향:** 공동창업자 후보 최소 1명의 실명(이니셜이라도)과 경력, 합류 조건을 명시. "합류 확정은 아니나 MOU/LOI 수준의 의향서를 확보했다"까지 가면 +3~5점. 당장 어렵다면 자문단·멘토 구성이라도 명시하여 "혼자가 아님"을 보여줘야 함.

2. **[기술 검증] AI 생성 결과물이 시뮬레이션(가상 데이터)** -- 제품의 핵심 가치가 "AI가 STAR 경력 기록을 자동 생성한다"인데, 실제 사용 데이터 기반 결과물이 없음. 이는 "맛집인데 음식 사진이 없다"와 같음. --> **권장 수정 방향:** 본인이 CareerLog를 1~2일 실제 사용하여 생성된 STAR 결과물 스크린샷을 첨부. Before(수동 기억 기반 이력서) vs After(CareerLog 자동 생성 경력 기록) 비교가 있으면 더 강력. 이것은 당장 실행 가능한 액션이므로 제출 전 반드시 수행.

3. **[사업화 계획] Free-to-Pro 전환 퍼널의 구체적 트리거 부재** -- "기록이 쌓이면 자연스럽게 유료 전환"은 희망사항. 구독 SaaS에서 전환율 5%를 달성하려면 명확한 friction point 설계가 필요. --> **권장 수정 방향:** 전환 트리거를 명확히 설계. 예: (1) 기록 3개월 이상 축적 시 "이직 준비도 리포트" 알림 → Pro 전환 유도, (2) Free에서 AI 생성 3회 사용 후 "이번 달 남은 횟수 0회" 시점이 전환 모멘트, (3) LinkedIn/원티드 내보내기를 Pro 전용으로 설정하여 이직 시점에 자연스럽게 결제 유도. 이런 퍼널 시나리오가 문서에 있어야 심사위원이 "이 사람은 비즈니스를 이해하고 있구나"라고 판단.

### 추가 개선 권장 사항 (치명적은 아니나 점수 상승 기여)

4. **[문서 완성도] "[확인 필요]" 태그, 미완 액션 아이템 정리** -- 제출 문서에 미완료 표시가 남아있으면 "준비가 안 된 지원자"로 인식됨. 창조경제혁신센터 접촉 계획도 "[창업자 확인 필요]"로 남아있는데, Pre-TIPS는 센터 추천이 필수이므로 이 항목이 미확정이면 지원 자체가 불가.

5. **[시장 분석] 설문 n=34의 한계 인정 + 보완 계획 명시** -- n=34는 통계적 유의성이 약함. 심사위원도 알고 있음. 차라리 "Pre-TIPS 기간 중 n=200 규모 설문 재실시 예정"이라고 명시하면, 현재 데이터의 약점을 인지하고 있다는 점에서 오히려 신뢰를 줌.

6. **[자금 계획] 팀 보강 비용 현실성 점검** -- 월 333만원으로 개발자 채용은 비현실적. 공동창업자라면 지분 기반이므로 급여 333만원이 아닌 생활비 보조 명목으로 재구성하거나, 파트타임 외주 개발자 활용으로 변경하는 것이 현실적.

## Pre-TIPS 특수 고려 사항

Pre-TIPS는 TIPS 진입을 위한 준비 과제이므로, 일반 창업지원사업과 달리 다음을 추가로 평가한다:

| 항목 | 현재 상태 | 평가 |
|------|----------|------|
| 창조경제혁신센터 추천 | 미확보 ("[확인 필요]") | 지원 불가 — 최우선 해결 필요 |
| TIPS 운영사 연계 가능성 | 구체적 운영사 미특정 | 약함 — 최소 2~3개 운영사 사전 접촉 권장 |
| 기술의 TIPS 적합성 | Win32 API + AI — 기술 난이도 중간 | 보통 — TIPS는 "기술 혁신"을 강조하므로, 특허 1건이라도 있으면 확연히 다름 |
| MVP 완성도 | v0.8 완성, GitHub 공개 | 강점 — 실행력 증명에 효과적 |
| 시장 검증 수준 | 설문 34명, 실사용자 0명 | 약함 — 베타 유저 최소 10명의 실사용 데이터 필요 |

---

> 본 심사평은 2026-03-26 기준 모의 심사이며, 실제 정부 심사위원의 판단과 다를 수 있습니다.
