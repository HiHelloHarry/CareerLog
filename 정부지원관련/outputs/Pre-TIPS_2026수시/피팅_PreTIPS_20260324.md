## 1. 문제-솔루션 적합성 (PSF)

### 해결하는 문제: "경력 3년치가 기억에만 있다"

직장인이 이직·연봉 협상·성과 평가를 앞두고 공통으로 겪는 문제가 있다. 지난 수 년간 실제로 무엇을 했는지 구체적 수치로 설명할 기록이 없다는 것이다.

- 이직 결심 후 경력 정리에 상당한 시간이 소요되며, 결국 직함과 회사명만 쓴 빈 이력서로 마무리된다.
- 연봉 협상·성과 평가 자리에서 수치 근거 없이 밀린 경험은 기획자·PM·디자이너 직군에서 반복된다.
- 문제는 이력서 작성 능력이 아니다. **기록되지 않은 경험**이 원인이다.

**기록이 생기지 않는 구조적 이유**

기록하려면 행동을 멈춰야 한다. 업무 중 별도로 기록하는 행위가 부담이기 때문에, 노션·옵시디언 같은 도구가 있어도 지속 사용되지 않는다.

**기존 솔루션이 채우지 못한 공백**

| 도구 | 한계 |
|------|------|
| 노션·옵시디언 | 기록 가능하나 구조화·자동화 없음. 지속 사용률 낮음 |
| 링크드인 | 수동 입력 필요. 업데이트 방치 패턴 고착 |
| AI 이력서 툴 (Resume.io, Rezi 등) | 작성 도움만 제공. 기억 자체가 없으면 소용없음 |
| 타임트래킹 툴 (Toggl 등) | 시간만 기록. "어떤 성과였는지" 해석 불가 |

**핵심 공백:** "경력을 자동으로 축적"하는 카테고리 자체가 현재 시장에 없다.

### 솔루션: CareerLog — PC가 알아서 기록하는 업무 블랙박스

CareerLog는 Windows 데스크톱 앱이다. 사용자가 별도로 입력하지 않아도 된다.

"업무 시작" 버튼 하나로 PC가 블랙박스로 변한다. 어떤 앱을 얼마나 썼는지, 어떤 프로젝트에 시간을 쏟았는지 10초마다 자동으로 기록된다. 업무가 끝나면 AI가 그 기록을 STAR 포맷 경력 기술서로 변환한다.

```
업무 시작 버튼 클릭
    ↓
PC 포그라운드 창 자동 감지 (10초 간격)
    ↓
타임라인 자동 생성 (앱별·시간순, 유휴 시간 제외)
    ↓
업무 종료 → 프로젝트 태깅 (선택)
    ↓
AI가 STAR 포맷 경력 기록 자동 생성
    ↓
경력 대시보드 누적 → 이직·협상·평가 시 꺼내 씀
```

사용자가 하는 일은 단 두 가지다 — 시작 버튼, 그리고 이직할 때 복사 붙여넣기.

**CareerLog가 이 문제에 적합한 이유**

기존 경쟁 솔루션은 모두 "이미 기억하고 있는 것"을 잘 쓰게 도와준다. CareerLog는 "기억할 필요 없게" 만든다. PSF가 성립하는 이유는 솔루션이 문제의 근본 원인(기록 행위의 부담)을 제거하기 때문이다.

## 2. 기술 독창성

### "OS-level 자동 감지 × AI 경력 기록 생성"의 결합

**혁신 1: OS-level 활동 자동 감지 (행동 개입 제로)**

기존 타임트래킹 서비스는 사용자가 타이머를 직접 시작·종료해야 한다. CareerLog는 Windows PowerShell + Win32 GetForegroundWindow() API를 직접 호출하여, 사용자 개입 없이 10초마다 포그라운드 창을 자동 감지한다.

- UIAutomationClient 방식의 창 제목 오염 버그를 Win32 API 직접 호출로 해결 — 기록 정확도 검증 완료
- 오픈소스 네이티브 모듈(active-win) 의존성 없이 구현하여 Electron 패키징 환경에서의 배포 안정성 확보
- 15분 Idle 자동 감지로 비활성 세션 오염 방지

**혁신 2: 맥락 인식 AI 경력 기록 생성**

PC 활동 데이터(앱 사용 패턴, 작업 유형, 프로젝트 태그) + 사용자 직군·직급·기술스택 프로필을 결합하여 Claude API가 STAR 포맷 경력 기록을 자동 생성한다.

출력 구조: `{situation, task, action, result, skills, metrics_detected, bullets}` — 이력서·포트폴리오·LinkedIn에 직접 활용 가능한 포맷

**두 혁신의 결합이 만드는 진입장벽**

```
"기록 자동화" 단독 → 타임트래킹 서비스와 차별화 불충분
"AI 이력서 생성" 단독 → 입력 데이터가 없으면 무용
CareerLog = 자동 수집 × AI 생성 → 시장에 없는 조합
```

**기술 완성도 (v0.6, 베타 배포 준비 완료)**

| 기술 구성 요소 | 완성 상태 |
|---|---|
| Win32 GetForegroundWindow() 기반 활동 감지 엔진 | 완료 |
| PowerShell 영구 프로세스 (10초 루프, spawn 1회) | 완료 |
| Idle 감지 (15분, powerMonitor.getSystemIdleTime()) | 완료 |
| Claude API STAR 프롬프트 엔지니어링 | 완료 |
| 직군·직급·스킬 프로필 프롬프트 주입 | 완료 |
| 커리어 대시보드 (히트맵, 스트릭, 인사이트) | 완료 |
| Windows x64 Squirrel 인스톨러 | 완료 |

**프라이버시 설계**

모든 데이터는 사용자 PC 로컬에만 저장된다. 외부 서버 전송 없음. 이는 기술 선택(서버리스 로컬 JSON 아키텍처)의 결과이며, AI 도구 사용에 따른 개인정보 유출 우려를 구조적으로 차단한다.

| 요인 | 내용 |
|------|------|

### 경쟁 포지셔닝

```
              [기록 자동화 높음]
                    │
                    │   ★ CareerLog
                    │     (AI 자동 축적 + 출력)
                    │
 [이력서 출력 약함] ──────┼────── [이력서 출력 강함]
                    │
        노션·옵시디언│         Resume.io · Rezi
        (수동 기록)  │         (1회성 작성 도움)
                    │
              [기록 자동화 낮음]
```

CareerLog는 "기록 자동화"와 "출력 품질"을 동시에 해결하는 포지션이며, 이 포지션에 해당하는 경쟁 서비스가 현재 국내에 없다.

### 목표 고객 세그먼트

**1순위 — 경력직 이직 예비자 (직장인 3~7년차)**
- 경력은 있으나 정리가 안 된 기획자·PM·디자이너·개발자
- "나중에 필요하겠다"는 인식은 있으나 기록 행위를 시작하지 못하는 층
- CareerLog의 핵심 심리 소구: "이직 도구"가 아니라 "경력 보험"

**2순위 — 신입 취준생**
- 인턴·프로젝트 경험을 체계적으로 기록하고 싶은 층

**3순위 — 프리랜서·N잡러**
- 다수 클라이언트 프로젝트를 관리하고 포트폴리오를 축적해야 하는 층

| 채널 | 대상 | 구체적 활동 |

| 기획·PM 커뮤니티 | PM·기획자 직군 | 기록 습관 관련 콘텐츠 + 도구 소개 |
| 개인 블로그·SNS | 커리어 관심 독자 | "경력 기록 자동화" 실사용 후기 |
| 지인 네트워크 | 초기 10~30명 베타 유저 | 직접 설치 파일 배포 |

**진입 장벽 낮추기 전략**

- Free 플랜 설계: 기록은 무제한으로 쌓이게 하되, 꺼내 쓸 때(내보내기·AI 생성)에 유료 전환
- 기록을 쌓은 사람일수록 이탈 비용이 높아지는 구조 → 장기 리텐션 기반

**B2C → B2B 확장 경로**

B2B(HR 담당자, 팀장) 진입은 B2C 트랙션 확보 후. MAU 5,000명 이상 도달 시 채용 플랫폼 파트너십(잡코리아·원티드 등) 접촉 전략 실행.

## 7. TIPS 연계 경로

Pre-TIPS는 TIPS 진입의 준비 단계다. CareerLog의 TIPS 진입 전략은 다음과 같다.

```
[현재] MVP v0.6 완성, 베타 배포 준비 완료
    ↓  Pre-TIPS 지원금 활용
[1~2개월] 베타 런칭 → D30 리텐션 40% 달성
    ↓
[3~4개월] MRR 500만원, 유료 전환 데이터 확보
```

### 사회적 가치

**직장인의 협상력 강화**
기록 없이 밀리는 연봉 협상·성과 평가 구조를 개선한다. 근로자가 자신의 기여를 수치로 입증할 수 있을 때 정당한 보상 협상이 가능해진다.

기록 능력이 뛰어난 사람뿐만 아니라 기록을 못하는 사람도 자동으로 경력이 쌓이는 구조를 제공한다. 기획자·디자이너·PM처럼 코드로 성과를 증명하기 어려운 직군의 경력 가시화에 기여한다.
