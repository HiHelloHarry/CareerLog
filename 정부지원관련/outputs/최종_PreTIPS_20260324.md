## 1. 기술 혁신성

### "업무 블랙박스" — 시장에 없는 기술 카테고리를 최초 구현

CareerLog는 두 가지 기술 혁신의 결합이다.

**혁신 1: OS-level 활동 자동 감지 (행동 개입 제로)**

기존 타임트래킹 서비스는 사용자가 타이머를 직접 시작·종료해야 한다. CareerLog는 Windows PowerShell + Win32 GetForegroundWindow() API를 직접 호출하여, 사용자 개입 없이 10초마다 포그라운드 창을 자동 감지한다.

- UIAutomationClient 방식의 창 제목 오염 버그를 Win32 API 직접 호출로 해결 — 기록 정확도 검증 완료
- 네이티브 모듈 의존성 없이 구현하여 패키징 및 배포 안정성 확보
- Idle 15분 자동 감지로 비활성 세션 오염 방지

**혁신 2: 맥락 인식 AI 경력 기록 생성 — STAR 프롬프트 설계**

**[수정 항목 — Before]**
> "PC 활동 데이터 + 사용자 직군·직급·기술스택 프로필을 결합하여 Claude Sonnet API가 STAR 포맷 경력 기록을 자동 생성한다" + 출력 구조만 언급.

**[수정 항목 — After]**

PC 활동 데이터(앱 사용 패턴, 작업 유형, 프로젝트 태그) + 사용자 직군·직급·기술스택 프로필을 결합하여 Claude Sonnet API가 STAR 포맷 경력 기록을 자동 생성한다.

**STAR 프롬프트 설계 핵심 원칙:**

```
[입력 레이어 1] 사용자 프로필
직군: PM | 직급: 5년차 시니어 | 기술스택: Figma, Jira, SQL

[입력 레이어 2] 세션 메타데이터
총 시간: 3h 22m | 프로젝트 태그: "결제 개선" | 작업 유형: 기획/커뮤니케이션

[입력 레이어 3] 앱 활동 타임라인
Figma(1h 20m) → Slack(45m) → Notion(35m) → Zoom(42m)
```

**STAR 출력 매핑 로직:**
- `situation`: 프로젝트 태그 + 앱 패턴으로 업무 맥락 추론 (예: Figma + Zoom → "UI 기획 및 이해관계자 리뷰")
- `task`: 직군·직급 기준 책임 영역 자동 서술
  - PM: "기획 주도 및 이해관계자 의사결정 조율"
  - 개발자: "기술 구현 및 아키텍처 의사결정"
  - 디자이너: "사용자 경험 설계 및 비주얼 디렉션"
- `action`: 앱별 활동을 능동형 동사로 변환 (Figma→"화면 설계", Jira→"이슈 관리 및 우선순위 조정", Zoom→"이해관계자 회의 진행")
- `result`: `metrics_detected` 필드 유무에 따라 분기 — 수치 발견 시 포함, 없으면 정성적 성과 서술
- `bullets`: 이력서 직접 사용 가능한 1~3줄 성과 요약 (STAR 압축본)

**개인화 차별화 포인트:** 동일한 앱 사용 시간도 직군에 따라 다른 관점으로 재해석. PM이 3시간 Figma를 쓰면 "UI 기획 주도", 개발자가 같은 시간 쓰면 "개발 스펙 검토 및 디자인 QA"로 서술이 달라진다. 이것이 기존 AI 이력서 툴(입력 기반)과의 근본적 차이다.

**오류 방지 설계:**
- 활동 로그 5분 미만: AI 생성 비활성화, "세션 시간이 너무 짧습니다" 안내
- 단일 앱 세션: AI 생성 시 단순 활동 반영 (복합 추론 생략)
- 이 두 조건이 할루시네이션 발생 구간을 사전 차단하여 출력 품질을 보장한다

**두 혁신의 결합이 만드는 진입장벽:**

```
"기록 자동화" 단독 → 타임트래킹 서비스와 차별화 불충분
"AI 이력서 생성" 단독 → 입력 데이터가 없으면 무용
CareerLog = 자동 수집 × AI 생성 → 시장에 없는 조합
```

## 4. TIPS 연계 가능성

### Pre-TIPS → TIPS 진입 로드맵

Pre-TIPS는 TIPS 진입의 준비 단계다. CareerLog의 TIPS 진입 전략은 다음과 같다.

```
[현재] MVP v0.6 완성, 베타 배포 준비 완료
    ↓  Pre-TIPS 지원금 활용
[1~2개월] 베타 런칭 → D30 리텐션 40% 달성
    ↓
[3~4개월] MRR 500만원, 유료 전환 데이터 확보
    ↓
[4~6개월] TIPS 운영사(VC/액셀러레이터) IR 피칭
    ↓  베타 지표 + 수익 데이터 제시
[6개월+] TIPS 운영사 투자 2억원 + R&D 최대 8억원 신청
```

### 창조경제혁신센터 관계 구축 계획

**[수정 항목 — Before]**
> "[확인 필요] 창조경제혁신센터 관계 구축 및 추천 확보 계획은 신청 전 구체화가 필요합니다."

**[수정 항목 — After]**

창조경제혁신센터 관계가 현재 없다는 사실을 인정한다. 이를 해소하기 위한 구체적 행동 계획은 다음과 같다.

| 단계 | 내용 | 시점 |
|---|---|---|
| 1단계 | 서울·경기 창조경제혁신센터 담당 MD 이메일 접촉 — 데모 요청 | 2026년 4월 중 |
| 2단계 | 혁신센터 주관 IR 데모데이 또는 스타트업 쇼케이스 일정 확인 및 참가 신청 | 2026년 4~5월 |
| 3단계 | 베타 D30 리텐션 데이터 확보 후 혁신센터 담당 MD 대상 정식 IR 피칭 | 2026년 5~6월 |
| 4단계 | 혁신센터 추천 확보 또는 연계 액셀러레이터 IR 진행 | 2026년 6~7월 |

피칭 적기는 베타 리텐션 데이터가 확보되는 2026년 5월 이후다. 아이디어만 있는 팀이 아닌 "동작하는 제품 + 초기 지표"를 가진 팀으로 혁신센터에 접근하는 것이 설득력이 높다.

[창업자 확인 필요: 현재 알고 있는 창조경제혁신센터 담당자 또는 연결 가능한 네트워크가 있다면 추가 기재]
