# 1. 왜 지금 이 팀이 이 문제를 풀어야 하는가

**"직장인 경력 기록 자동화"는 5년 안에 반드시 나올 시장이다. 우리는 이미 만들었다.**

국내 사무·전문직 PC 기반 임금근로자는 약 840만 명이다. 이 중 이직 예비자 및 AI 도구 수용 의향자를 보수적으로 40%로 잡으면 약 132만 명이 잠재 유료 고객이다. 월 9,900원 구독 기준 SAM은 연 1,583억원이며, 이직·채용 플랫폼과의 제휴까지 확장하면 TAM은 연 1조원 이상이다.

> 출처 근거: 840만 명은 통계청 경제활동인구조사(2025) 사무 및 관련 종사자 + 전문가 및 관련 종사자 합산 기준. 이직 예비자 비율(40%)은 잡코리아·사람인 연간 이직 의향 조사(이직 의향자 비율 38~42% 구간, 2023~2024 평균) 참조. [창업자 확인 필요: 최신 잡코리아/사람인 설문 조사 원본 URL 인용 추가]

기존 솔루션(노션, 링크드인, 타임트래킹)은 모두 "이미 기억하는 것"을 잘 쓰게 도와줄 뿐이다. **기억 자체를 자동으로 축적하는 서비스는 지금까지 존재하지 않았다.** CareerLog는 이 공백을 최초로 점유한다.

# 2. 제품: 무엇을 만들었는가

CareerLog는 Windows 데스크톱 앱이다. 사용자 동작은 두 가지뿐이다.
1. 업무 시작 버튼
2. 이직할 때 복사 붙여넣기

그 사이 PC 포그라운드 창을 10초마다 자동 감지하고, 업무 종료 시 Claude AI가 STAR 포맷 경력 기록을 자동 생성한다.

**기술 핵심 — 두 혁신의 결합:**

**혁신 1: OS-level 활동 자동 감지**
PowerShell + Win32 GetForegroundWindow() API 직접 호출로 네이티브 모듈 의존성 없이 구현. UIAutomationClient 방식의 창 제목 오염 버그를 Win32 API 방식으로 해결하여 기록 정확도를 확보했다.

**혁신 2: 맥락 인식 STAR 프롬프트 설계**
PC 활동 로그(앱명, 창 제목 타임라인, 작업 유형 태그, 세션 총 시간) + 사용자 직군·직급·기술스택 프로필을 Claude Sonnet API에 주입하여 STAR 포맷 경력 기록을 자동 생성한다.

**STAR 프롬프트 설계 원칙:**
- **입력 구조**: `[직군] [직급] [기술스택] + [앱별 활동 타임라인] + [프로젝트 태그] + [작업 유형(기획/개발/QA 등 8개 카테고리)]`
- **출력 구조**: `{situation, task, action, result, skills, metrics_detected, bullets}`
  - `situation`: 어떤 프로젝트/맥락에서 발생한 업무인지 (앱 패턴 + 태그로 추론)
  - `task`: 직군·직급에 맞는 책임 영역 기술 (PM이면 "기획 및 의사결정", 개발자면 "구현 및 기술 문제 해결")
  - `action`: 실제 수행한 활동을 능동형 동사 중심으로 서술 (앱 타임라인 데이터 기반)
  - `result`: `metrics_detected`가 있으면 수치 포함, 없으면 정성적 성과 서술
  - `bullets`: 이력서 복사 붙여넣기 가능한 1~3줄 요약
- **개인화 핵심**: 동일한 활동 로그라도 직군에 따라 다른 관점의 경력 기록 생성. PM에게는 "기획 주도", 개발자에게는 "기술 구현" 관점으로 서술이 달라짐
- **오류 케이스 처리**: 활동 로그가 5분 미만이거나 앱 다양성이 1개뿐인 경우, AI 생성을 비활성화하고 "기록이 충분하지 않습니다" 안내 반환 (할루시네이션 방지)

[창업자 확인 필요: JD 매칭 알고리즘 기술 설계 초안 — 현재 미구현 기능으로 기술 로드맵에만 포함할지, 구체적 설계를 별도 섹션으로 추가할지 결정 필요]

**현재 상태 (v0.6, 투자 유치 시점):**

| 기능 | 상태 |
|---|---|
| PC 활동 자동 감지 (10초 폴링, Idle 15분 감지) | 완료 |
| 세션 시작/종료, 프로젝트 태깅 | 완료 |
| AI STAR 경력 기록 자동 생성 (Claude API) | 완료 |
| 커리어 대시보드 (히트맵, 스트릭, 인사이트) | 완료 |
| Markdown/HTML 내보내기, 클립보드 복사 | 완료 |
| Windows x64 Setup.exe 인스톨러 | 완료 |

MVP는 이미 동작한다. 투자금은 "만들기"가 아닌 "성장"에 쓰인다.

# 5. 비즈니스 모델 및 수익 전망

**B2C 구독 → B2B SaaS → 전략적 인수**

| 단계 | BM | 목표 지표 |
|---|---|---|
| Phase 1 (4~5월) | 베타 무료 배포 | D30 리텐션 40% |
| Phase 2 (6~7월) | Pro 월 9,900원 / Premium 19,900원 | MRR 500만원 |
| Phase 3 (8~11월) | Mac 확장, 콘텐츠 마케팅 | MAU 5,000명 |
| Phase 4~5 (12개월+) | JD 매칭, 채용 플랫폼 파트너십, B2B 팀 플랜 | ARR 6~24억원 |

**엑시트 내러티브:** 잡코리아(2021년 카카오 1.7조 인수), 원티드랩(시총 5,000억), 사람인 등 국내 커리어테크는 대형 M&A가 반복되는 시장이다. CareerLog는 이들이 아직 보유하지 못한 "업무 기록 자동화 레이어"를 선점한다. B2C 사용자 데이터 + 이직 행동 데이터가 확보되는 시점에 전략적 인수 가치가 극대화된다.

# 6. 글로벌 확장 전략

- **1단계 (국내 검증):** Windows x64, B2C 구독 리텐션 확보
- **2단계 (일본):** 일본은 잦은 이직 문화 + Windows PC 점유율 높음. 별도 로컬라이제이션 없이 UI 번역으로 진입 가능
- **3단계 (동남아·영어권):** 활동 감지 엔진은 OS-level이므로 언어 독립적. 영어권 LinkedIn 사용자 공략 가능
