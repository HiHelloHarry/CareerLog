## 핵심 피드백 요약

### 강점

1. **MVP 완성 상태로 지원** — 예비창업 단계에서 Windows x64 인스톨러 배포까지 완성한 것은 같은 단계 지원자 대비 명확한 차별점이다. "만들 것이다"가 아닌 "만들었다"는 것이 3인 모두의 공통된 긍정 평가 요소다.

2. **시장 근거의 투명성** — TAM/SAM 수치에 출처를 명시하고 추정임을 스스로 밝혔다. AI API 비용도 역산 근거를 제시했다. 과장된 시장 규모를 주장하는 대다수 계획서와 차별화된다.

3. **문제-창업자 일치성** — 창업자 본인이 문제의 당사자이면서 기술적 해결자라는 점이 계획서 전반에서 자연스럽게 드러난다. 진정성 측면에서 3인 모두 높은 평가를 줬다.

### 보완 필수 항목

1. **[고객 검증 부재]** 사업계획서 전체에 실제 사용자 인터뷰, 설문, 또는 베타 피드백이 단 한 건도 없다. 창업자의 경험이 구조적 문제임을 입증하는 외부 목소리가 없으면 주관적 가정에 머문다. → 잠재 타겟 10~20명 인터뷰를 진행하고 "이직 준비 시 경력 정리에 얼마나 시간이 걸렸는가", "자동 기록 툴이 있다면 쓸 의향이 있는가" 수준의 1차 데이터를 확보해 계획서에 포함할 것.

2. **[솔루션 품질 수치 부재]** PC 활동 자동 감지의 정확도(오탐·미탐률)와 AI STAR 생성 품질에 대한 수치적 근거가 없다. "잘 작동한다"는 주장을 심사위원 입장에서 검증할 방법이 없다. → 창업자 본인의 사용 기록 기반이라도 "1개월 사용 시 X개 세션, 노이즈 Y건 발생, STAR 생성 품질 자가 평가" 수준의 수치를 제시해 신뢰 근거를 보완할 것.

3. **[단독 창업 리스크]** 결제 시스템 외주, Mac 개발, 그로스 마케팅 직접 운영, 베타 유저 피드백 반영을 1인이 Phase 1~2 구간에 동시에 수행하는 것은 실행 과부하다. 외주 개발자 관리·코드 리뷰 계획도 없고, 창업자 본인의 생존 자금 계획도 빠졌다. → 공동창업자 탐색을 공식 계획에 포함하거나, 최소한 기술 외주 관리 방법론과 창업자 인건비 계획을 자금 계획에 추가할 것.
