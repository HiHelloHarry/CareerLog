## 1. 창업 아이템 개요

**아이템명:** CareerLog — 직장인 경력 자동 기록 서비스

**한 줄 정의:** 업무 중 PC가 알아서 경력을 쌓고, AI가 이력서용 경력 기록으로 변환해주는 Windows 데스크톱 앱

## 2. 문제 정의

### 창업자가 직접 경험한 구조적 문제

직장인은 일을 하면서도 경력을 쌓지 못한다. 이것은 의지의 문제가 아니라, 구조의 문제다.

이직을 결심한 직장인 3~7년차의 패턴은 공통적이다.

> "어떤 프로젝트를 했지?" → 기억이 없다 → 정리 포기 → 이직 기회 소멸

기억하려면 일을 멈춰야 하고, 일하는 동안에는 기록할 수 없다. 이 구조적 간극이 경력 기록 실패의 본질이다.

**창업자 본인의 경험:** 기획·QA·PM 멀티롤로 실무를 수행하며 이 문제를 반복 경험했다. 경력은 쌓였지만 정리는 되어 있지 않았고, 이직 준비 시 활용 가능한 형태의 기록이 없었다. 이 문제를 해결하기 위해 CareerLog를 기획하고 직접 개발했다.

[창업자 확인 필요: 베타 모집 전 이직 경험 직장인 10~20명 대상 인터뷰 수행 후 핵심 인용구 삽입 권장 — "이직 준비 시 경력 정리에 얼마나 시간이 걸렸는가", "자동 기록 툴이 있다면 쓸 의향이 있는가" 수준의 1차 데이터]

### 시장 규모로 확인되는 문제의 범위

- 국내 경영·회계 관련 사무직 임금근로자는 **426만 2천 명** (통계청 2024년 상반기 지역별고용조사, 직업중분류 전체 1위 항목)으로, PC 기반 업무 종사자 중 경력 기록 수요가 가장 높은 직군이다.
- 20~40대 정규직 근로자 **69.5%가 이직을 고려** 중이다 (한국경영자총협회 「근로자 이직 트렌드 조사」, 2024.04, 글로벌리서치 위탁, 전국 20~40대 정규직 1,500명). 이직 고려율이 구조적으로 높을수록 경력 관리 수요도 함께 확대된다.
- 국내 취업자의 업무용 생성형 AI 활용률은 **51.8%** (한국은행 BOK 이슈노트 제2025-22호, 2025.08)로, 미국(26.5%)의 약 2배 수준이며 AI 기반 업무 도구에 대한 수용도가 높다.

### 기존 솔루션의 공통 맹점

모든 기존 도구는 "이미 기억하는 것"을 잘 쓰도록 도와줄 뿐이다. 기억 자체를 자동으로 쌓아주는 서비스는 없다.

| 기존 솔루션 | 방식 | 근본 한계 |
|---|---|---|
| 노션 / 메모 앱 | 직접 입력 | 기록하려면 업무를 멈춰야 함 |
| 링크드인 | 직접 작성 | 과거 회고 방식, 기억 의존 |
| 타임트래킹 (Toggl 등) | 수동 타이머 | 시작/종료 직접 입력 필요. 성과 해석 불가 |
| AI 이력서 툴 (Resume.io, Rezi 등) | 입력 기반 생성 | 입력할 내용 자체가 없으면 무용 |
| **CareerLog** | **PC가 알아서 자동 감지** | — |

**핵심 공백:** "경력을 자동으로 축적"하는 카테고리 자체가 국내외 시장에 없다.

## 3. 솔루션 및 혁신성

### "업무 블랙박스" — 일하는 동안 PC가 알아서 경력을 쌓는다

CareerLog는 사용자가 업무 중 의식적으로 무언가를 입력하지 않아도 된다. 사용자가 하는 것은 단 두 가지다.

1. **업무 시작 버튼을 누른다**
2. **이직할 때 복사해서 붙여넣는다**

그 사이의 모든 과정은 CareerLog가 자동으로 처리한다.

```
업무 시작 버튼
    ↓
PC 포그라운드 창 10초마다 자동 감지 (PowerShell + Win32 GetForegroundWindow API)
    ↓
앱별 활동 타임라인 자동 생성 (15분 이상 자리 비움은 자동 제외)
    ↓
업무 종료 버튼
    ↓
AI가 STAR 포맷 경력 기록 자동 생성
    ↓
Markdown / HTML 내보내기, 클립보드 복사
```

### 혁신성 및 차별성

**1. 입력 없는 자동 기록**

시중의 모든 유사 도구는 사용자의 능동적 입력을 전제로 한다. CareerLog는 PC 활동을 수동 개입 없이 자동 감지한다. 국내에 이 컨셉의 서비스는 현재 존재하지 않는다.

**2. 로컬 우선 프라이버시 설계**

활동 데이터는 전량 사용자 PC에만 저장된다. AI 경력 기록 생성 시에도 원본 활동 데이터는 외부 서버로 전송되지 않는다. 직장인이 업무 기록 툴을 기피하는 주요 이유(보안·프라이버시 우려)를 구조적으로 해소한다.

**3. 기술적 안정성**

외부 네이티브 모듈 없이 Windows Win32 API를 PowerShell을 통해 직접 호출하는 독자적 구현 방식을 채택했다. 오픈소스 의존성 없이 Electron 패키징 환경에서 안정적으로 동작하며, 배포 후 의존성 오류 가능성을 원천 차단한다.

UIAutomationClient 방식의 창 제목 오염 버그(타 앱 제목이 교차 기록되는 문제)를 직접 발견하고 Win32 GetForegroundWindow() API 방식으로 전환해 해결했다. 이 버그 발견과 수정은 동일 환경에서 동작하는 오픈소스 대비 기록 정확도 우위의 근거다.

**4. AI 개인화 구조**

사용자의 직군·직급·기술스택을 AI 프롬프트에 자동 주입하여, 동일한 활동 기록이라도 "기획자 시각"과 "개발자 시각"으로 각각 다른 경력 기록을 생성한다.

### 글로벌 경쟁 포지셔닝

| 서비스 | 자동 기록 | AI 경력 변환 | 로컬 저장 | 한국어 지원 | 국내 이직 최적화 |
|---|---|---|---|---|---|
| **CareerLog** | O | O (STAR 포맷) | O | O | O |
| MS Recall | O (화면 전체 캡처) | X | 온디바이스 (일부 전송 논란) | X | X |
| Rewind AI | O (화면 캡처) | X | O (Mac 전용) | X | X |
| Notion·Obsidian | X (수동 입력) | X | 선택 가능 | O | X |
| Resume.io·Rezi | X | O (작성 보조) | X | X | X |

CareerLog는 **한국 직장인의 이직 문서(경력기술서·STAR)에 특화된 유일한 로컬 저장 자동화 서비스**다.

MS Recall은 화면을 통째로 저장하는 방식으로 개인정보 논란이 지속되고 있으며, 국내 이직 시장에 특화된 AI 경력 기록 변환 기능이 없다. Rewind AI는 Mac 전용이며 경력 문서 생성을 지원하지 않는다.

## 4. 시장 분석

### 타겟 시장

**1순위 타겟: 이직을 준비하거나 예비하는 직장인 3~7년차**
- 기획자, PM, 디자이너, 개발자 등 PC 기반 전문직
- 경력은 쌓였으나 정리된 기록이 없는 상태
- Windows PC 사용 비중 높고, 로컬 저장 방식에 대한 신뢰도 높음

**2순위 타겟:**
- 신입 취준생: 인턴·프로젝트 경험을 포트폴리오로 전환하는 수단
- 프리랜서·N잡러: 다수 프로젝트의 실적을 클라이언트별로 정리

### 시장 규모 (TAM-SAM-SOM)

| 구분 | 정의 | 규모 |
|---|---|---|
| **TAM** | 국내 경영·회계 관련 사무직 임금근로자 (PC 기반 업무) | **426만 2천 명** (통계청 2024년 상반기 지역별고용조사) × 연 119,880원 = **약 5,109억 원/년** (추정) |
| **SAM** | 이직을 고려 중인 사무직 근로자 (이직 고려율 69.5% 적용) | **약 296만 명** (추정) × 연 119,880원 = **약 3,549억 원/년** (추정) |
| **SOM** | 초기 18개월 내 도달 목표 (커뮤니티·콘텐츠 채널 기반) | 유료 5,000~20,000명 목표, ARR 약 6~24억 원 (추정) |

> TAM 출처: 통계청 2024년 상반기 지역별고용조사 취업자 직업별 특성 (2024.10.8 발표). 경영·회계 관련 사무직 426만 2천 명은 직업중분류 전체 1위 항목.
> SAM 출처: 한국경영자총협회 「근로자 이직 트렌드 조사」(2024.04, 글로벌리서치 위탁, 전국 20~40대 정규직 1,500명) — 69.5%가 이직 고려 중. SAM = TAM × 69.5% 적용 (추정).
> SOM 목표는 내부 추정이며, 베타 유저 데이터 확보 후 조정 예정.

## 6. 사업화 계획 (지원 기간 내 실행 계획)

예비창업패키지 지원금은 아이디어를 제품으로 만드는 데 쓰는 것이 아니다. 제품은 이미 완성됐다. **이 자금은 제품을 시장에 연결하고 검증하는 데 사용한다.**

### Phase 1 — 베타 런칭 및 리텐션 검증 (1~6개월)

**핵심 목표:** 베타 유저 100명 모집, D30 리텐션 40% 달성

## 9. 기대 효과 및 사회적 가치

### 직장인의 경력 자본 격차 해소

이직 시장에서 경력 정리의 품질은 현재 개인의 글쓰기 능력과 시간 여유에 따라 결정된다. 업무가 바쁠수록, 글쓰기가 서툴수록, 경력 기록에서 불리해지는 구조다.

CareerLog는 일하는 동안 자동으로 경력을 축적함으로써, 경력 정리에 투입할 시간이 없는 직장인도 동일한 출발선에 설 수 있도록 한다.

- 이직 준비 기간 단축 → 노동 시장 유동성 증가
- AI 기반 경력 기록의 민주화 → 기획자·디자이너·PM처럼 코드로 성과를 증명하기 어려운 직군의 경력 가시화
- 로컬 저장 방식으로 직장인 개인정보를 구조적으로 보호

## 10. 창업 의지

저는 이 문제를 직접 겪었고, 직접 해결했습니다.

기획자·QA·PM으로 일하면서 경력은 쌓였지만 정리된 기록이 없었습니다. 이직을 준비할 때마다 "내가 뭘 했지?"라는 질문 앞에서 막혔고, 그 시간을 반복할 수 없다고 판단해 직접 만들기 시작했습니다.

아이디어에서 출발해 기술 스택 선정, 개발, QA, 배포용 인스톨러 제작까지 1인으로 완성했습니다. 제품은 이미 작동합니다. 예비창업패키지 지원금으로 베타 사용자를 확보하고, 리텐션 데이터를 쌓고, 이 제품이 시장에서 필요한 것임을 증명하겠습니다.
