## □ 창업 아이템 개요(요약)

### 명칭
> CareerLog

### 범주
> 업무 활동 자동 기록 기반 커리어 관리 데스크톱 서비스

### 아이템 개요
> 업무 시작/종료 두 버튼만 누르면, 어떤 앱을 얼마나 쓰고 어떤 프로젝트에 시간을 투자했는지 자동으로 기록된다. 귀찮은 입력 없이 커리어가 매일 쌓이고, 대시보드에서 내 업무 패턴을 한눈에 확인할 수 있다. 이렇게 쌓인 기록은 필요할 때 AI가 STAR 경력기술서로 자동 변환해준다. 모든 데이터는 사용자 PC에만 저장(로컬 우선 설계)되어 보안 우려를 해소한다.
>
> 핵심 기능: (1) Win32 API 기반 업무 활동 자동 기록 (10초 폴링, 앱별 파서 17개) — 귀찮음 제로, (2) 커리어 대시보드 (히트맵, 스트릭, 인사이트) — 체계적 커리어 설계, (3) AI STAR 경력 기록 변환 (직군·직급 개인화) — 필요할 때 꺼내 씀, (4) Markdown/HTML 내보내기·클립보드 복사 — 즉시 출력
>
> 고객 제공 혜택: 매일 2분 투자(업무 시작·종료 버튼)만으로 체계적 커리어 관리. 쌓인 기록은 이직 시 AI가 경력기술서로 자동 변환하여 "며칠"의 정리 시간을 "버튼 한 번"으로 단축. 자체 설문(n=34)에서 응답자 76%가 긍정적 사용 의향, 71%가 유료 지불 의사 확인.

### 문제 인식 (Problem)
> 직장인은 경력 관리가 중요한 걸 알지만, 귀찮아서 아예 안 한다. 자체 설문(n=34)에서 88%가 상시 경력 관리를 하지 않는다고 답했고, 그 이유는 "시간이 없어서"가 아니라 "기록 자체가 귀찮아서"다. 이직 준비 시 "어떤 프로젝트를 했지?"라는 질문 앞에서 62%가 하루 이상을 소비하며, 성과를 수치로 표현하기 어려운 것이 1위 페인포인트(59%)다. 기존 노션·엑셀·AI 이력서 툴은 모두 "이미 기억하는 것"을 입력해야 하며, 입력의 귀찮음을 제거하고 기록 자체를 자동으로 쌓아주는 서비스는 국내외에 없다.

### 실현 가능성 (Solution)
> MVP v0.8 개발 완료 상태. 사용자는 업무 시작·종료 두 버튼만 누르면 커리어가 자동으로 쌓이고, 필요할 때 AI가 STAR 경력기술서로 변환해준다. Windows x64 Setup.exe 인스톨러 배포 가능. GitHub 공개 저장소에서 전체 소스 코드·커밋 이력 확인 가능(https://github.com/HiHelloHarry/CareerLog). 협약기간 내 (1) 베타 유저 100명 확보 및 리텐션 검증, (2) B2C 결제 시스템 구축, (3) Mac 버전 개발을 최종 산출물로 완성할 계획.

### 성장전략 (Scale-up)
> 매일 업무 시작·종료와 함께 사용하는 습관 도구로 자연스럽게 일상 루틴에 편입 → 기록이 쌓일수록 이탈 비용 상승 → Pro 구독 런칭(월 4,900원) → 채용 플랫폼 파트너십 → B2B 확장. 자체 설문에서 유료 의향 71%, 유료 의향 응답자 중 월 5,000원 이하 수용 71%(17/24명), 전체 기준 50% 확인. BEP 유료 구독자 약 300명 시점(서버·API 비용 기준, 초기 무급 운영 전제). 국내 HR테크 시장 CAGR 7.55%(IMARC 2024), "자동 커리어 관리" 카테고리 선점 기회.

### 팀 구성 (Team)
> 대표자가 기획·개발·QA·배포 전 과정을 주도. 기획/QA/PM 멀티롤 실무 경력 보유. UI 기획 및 디자인 담당 팀원 합류 완료, CTO(풀스택 개발) 영입 예정('26.하반기).

### 이미지
> (GitHub Releases 스크린샷, 업무 시작/종료 화면, 타임라인 자동 기록 화면, 커리어 대시보드 캡처 삽입)

## 1. 문제 인식 (Problem) — 창업 아이템의 필요성

### 국내·외 시장 현황 및 문제점

**1) 직장인 경력 관리의 구조적 실패 — 기록이 귀찮은 구조적 이유**

직장인은 경력 관리가 중요한 걸 알면서도 하지 않는다. 이것은 의지의 문제가 아니라, 기록 자체가 귀찮은 구조의 문제다.

- 국내 경영·회계 관련 사무직 임금근로자 **426만 2천 명** (통계청 2024년 상반기 지역별고용조사, 직업중분류 전체 1위)
- 20~40대 정규직 근로자 **69.5%가 이직 고려** 중 (한국경영자총협회 「근로자 이직 트렌드 조사」, 2024.04, 글로벌리서치 위탁, n=1,500)
- 국내 취업자 업무용 생성형 AI 활용률 **51.8%** (한국은행 BOK 이슈노트 제2025-22호, 2025.08), 미국(26.5%) 대비 약 2배

**2) 잠재 고객 설문으로 확인된 문제의 실체 (n=34, 2026.03.25, 자체 Google Forms 설문)**

| 핵심 지표 | 수치 | 시사점 |
|---|---|---|
| 이직할 때만 or 분기 이하로 경력 정리 | **88%** | 귀찮아서 안 하는 것이 핵심 — "자동 기록" 가치 명확 |
| 이력서 작성 시 "수치 표현 어렵다" | **59%** (1위) | 수치 자동 추출 기능 수요 |
| 경력 정리에 하루 이상 소요 | **62%** | 시간 절약 가치 검증 |
| "바로 써보고 싶다" + "관심 있다" | **76%** | 서비스 수용도 높음 |
| 유료 지불 의사 | **71%** | WTP 검증 |
| 핵심 연차 3~5년차 | **41%** (최다) | 타겟 프로파일 일치 |
| 이직 관련 활성 상태 | **76%** | 경총 69.5%와 교차 검증 |

> 설문 대상: 기획/PM(26%), 디자인(24%), 개발(18%), QA(9%) 등 9개 직군, 경력 1년 미만~10년 이상 분포. Google Forms 원본 데이터(CSV) 보유.

**3) 기존 솔루션의 공통 맹점**

| 기존 솔루션 | 방식 | 근본 한계 |
|---|---|---|
| 노션 / 메모 앱 | 직접 입력 | 기록하려면 업무를 멈춰야 함 → 귀찮아서 안 하게 됨 |
| 링크드인 | 직접 작성 | 과거 회고 방식, 기억 의존 → 매일 쓸 이유 없음 |
| Toggl 등 타임트래킹 | 수동 타이머 | 매번 프로젝트 선택·시작/종료 입력 → 습관화 실패 |
| Resume.io, Rezi 등 | 입력 기반 생성 | 입력할 내용 자체가 없으면 무용 → 기록이 선행돼야 함 |

설문에서도 현재 경력 기록 방법은 "노션/메모(38%)", "엑셀(32%)", "기억에 의존(18%)" 순으로, 자동화 대안 없이 수동 방식에 머물러 있음이 확인됐다.

**핵심 공백:** "귀찮음 제로로 매일 커리어가 자동으로 쌓이는" 카테고리 자체가 국내외 시장에 없다. 기존 도구는 모두 능동적 입력을 전제로 하기에, 88%가 경력 관리를 포기하는 귀찮음의 구조를 해결하지 못한다.

### 문제 해결을 위한 창업 아이템의 개발 필요성

기존 도구는 "이미 기억하는 것"을 잘 쓰도록 도와줄 뿐이다. CareerLog는 귀찮음을 구조적으로 제거한다. 업무 시작·종료 두 버튼만 누르면 PC 활동이 자동으로 기록되고, 대시보드에서 내 업무 패턴을 한눈에 확인할 수 있다. 이렇게 매일 쌓인 기록은 이직이 필요할 때 AI가 STAR 경력기술서로 변환해준다. 설문 결과 "업무 활동 자동 기록"이 가장 끌리는 기능 1위(47%)를 차지했으며, 이 문제의 해결이 돈을 낼 만한 가치인지에 대해 71%가 유료 지불 의사를 보였다.

**사용자가 하는 일은 단 두 가지:** 업무 시작, 업무 종료 — 이 두 버튼만으로 매일 커리어가 쌓인다. 이직할 때? 쌓인 기록에서 AI가 이력서를 만들어준다.

**솔루션 플로우:**
1. 업무 시작 버튼 클릭
2. PC 활동 자동 감지 (10초 간격, 앱별 파서 17개)
3. 타임라인 + 대시보드에 매일 커리어가 쌓임
4. 업무 종료 버튼 클릭 → 오늘의 업무 요약 확인
5. 필요할 때 AI가 STAR 경력 기록으로 자동 변환
6. 이력서·LinkedIn에 복사 붙여넣기

### 창업 아이템의 차별성 및 경쟁력 확보 전략

**1. 귀찮음 제로 — 버튼 두 개로 매일 커리어가 쌓인다** (국내 유일)
시중 모든 도구는 사용자의 능동적 입력을 전제로 한다. CareerLog는 업무 시작·종료 두 버튼만으로 PC 활동을 자동 감지하여 매일 커리어를 쌓아준다. 설문에서 "업무 활동 자동 기록"이 가장 끌리는 기능 1위(47%). 매일 업무 시작/종료와 함께 사용하는 습관 도구로, 자연스럽게 일상 루틴에 편입되어 높은 DAU를 확보할 수 있는 구조.

**2. 로컬 우선 프라이버시 설계**
활동 데이터는 전량 사용자 PC에만 저장. 설문 자유 의견에서 보안 우려가 15%로 반복 등장했으며, CareerLog의 로컬 저장 + 앱 블랙리스트 + URL depth 제한 + 민감 도메인 차단 기능이 이 우려에 직접 대응.

**3. 기록 정확도 기술적 해자(Moat)**
Win32 API 직접 호출(네이티브 모듈 의존 없음), 앱별 파서 17개, 브라우저 URL 추출, 키보드·마우스 활동 감지, 작업 흐름(Flow) 감지. UIAutomation 방식의 창 제목 오염 버그를 직접 발견·수정한 독자적 구현.

CareerLog는 기록이 쌓일수록 유료 전환 압력이 자연스럽게 높아지는 구조다.

1. **기록 축적 효과:** Free 유저는 매일 기록이 쌓이지만, 세션 20개·AI 생성 월 3회 제한에 도달하면 "쌓아둔 기록을 꺼내 쓸 수 없는" 상태가 된다. 사용 기간이 길수록 축적된 기록의 가치가 커지고, 유료 전환 동기가 강해진다.

## 4. 팀 구성 (Team) — 대표자 및 팀원 구성 계획

### 창업 동기

창업은 다른 누군가의 응원보다 본인의 확신이 나아갈 수 있는 원동력을 준다고 생각합니다. "이것보다 잘 만들겠다"는 확신으로 게임 개발 직업을 택했고, 창업 팀원을 모으고 싶다는 생각에 팀 매칭 웹페이지를 직접 설계한 적이 있습니다. 행동의 시작점은 언제나 "직접 겪은 문제"였습니다.

AI를 활용한 1인 창업이 가능해진 시대에 AI 공부를 하던 중, 이직이 절실해졌습니다. 그런데 2년간 회사를 다니며 누구보다 열심히 일했다고 생각했는데, **"내가 뭘 했는지 기억이 나지 않았습니다."** 분명 그 누구보다 열심히 일했는데 기억이 나지 않는다는 사실이 황당했고, 기록할 시간도 없이 열심히 살아온 건데 하는 생각에 억울하기도 했습니다.

CareerLog는 이 억울함에서 출발했습니다.

> **"당신의 커리어를 기록하여 지켜드립니다"** — 개인의 니즈에서 시작하여,
> **"경력 기록 같은 사소한 일에 신경 쓰지 말고 본업에 온 힘을 쏟을 수 있도록"** — 조금 더 높은 기여로 이어지고,
> **"열심히 일한 사람이 정당하게 인정받는 사회를 만들겠습니다"** — 라는 사회적 기여로 마무리 짓고 싶습니다. 기록 능력이나 글쓰기 실력이 아니라, 실제로 일한 만큼 경력이 증명되는 구조를 만드는 것. 그것이 CareerLog가 사회에 기여하는 방식입니다.

결국 창업 아이템이란, 전략도 중요하지만 **"내가 무조건 사용할 서비스인가"**에서 시작한다고 생각합니다. CareerLog는 창업자 본인이 가장 절실하게 필요로 하는 서비스이며, 그렇기에 MVP를 단독으로 완성할 수 있었습니다.
