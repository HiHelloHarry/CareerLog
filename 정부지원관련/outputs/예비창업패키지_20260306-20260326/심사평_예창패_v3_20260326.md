## 핵심 피드백 요약

### 강점
1. **제품 완성도가 예비창업 수준을 초과한다.** v0.7~v0.8 고도화 완료, GitHub 소스/커밋/인스톨러 검증 가능. 기획서만 들고 오는 대부분의 지원자 대비 확실한 차별점.
2. **설문 데이터 활용이 체계적이고, 이전 심사의 수치 오류가 교정됐다.** 유료 의향 71%(합계 100%), Phase 2 MAU 3,000 + MRR 100만(산술 일치). 보안 우려(15%)를 솔직히 제시.
3. **메시징 리프레이밍이 효과적이다.** 버튼 두 개로 매일 커리어가 쌓인다가 전체를 관통. 설문 1위(자동 기록 47%)와 정렬. STAR가 메인으로 읽히는 구간 없음.

### 보완 필수 항목

1. **[팀 역량] 1인 팀 리스크 -- 3인 모두 10~11점/15점으로 최대 감점 요인**
   - 문제점: 후보 2명과 탐색적 대화가 이전 버전에서 변화 없음. 결제 외주 유지보수 공백.
   - 권장: 역량/조건 구체화, 합류 실패 시 플랜 B(외주 장기 계약 전환), 참여 의향 확인 노력.

2. **[시장성] Phase 3 ARR 6억(MRR 500만) 산술 근거 부족 + SOM-ARR 혼재**
   - 문제점: MAU 10,000 x 7% = 700명 x 4,900 = MRR 343만(목표 500만과 불일치). 또한 SOM 표에서 유료 약 1,000명 + ARR 약 6억이라 했으나, 1,000 x 58,800 = 5,880만(약 0.6억)으로 산술 불일치. SOM의 1,000명과 ARR 6억이 다른 시점의 수치가 혼재된 것으로 보임.
   - 권장: (a) ARPU 상승 근거(Premium 믹스 + B2B) 추가, (b) MRR 하향, (c) SOM 표의 인원수와 ARR 정합성 확보.

3. **[사업화] 리텐션 목표 경험적 근거 미약**
   - 문제점: D30 40%의 벤치마크 없음. 이탈 비용이 높아진다는 논리적이나 미검증.
   - 권장: 유사 습관 앱 벤치마크, 창업자 본인 사용 데이터, 또는 내부 목표이며 베타 실측 후 조정으로 톤 조정.

### 추가 개선 권장 (가점)

4. **지재권 전략 한 줄:** Flow 감지 알고리즘 방어적 특허 출원 검토 중.
5. **자동 시작 로드맵:** PC 부팅 시 자동 세션 시작 옵션 -- 습관 도구 가설 강화.
