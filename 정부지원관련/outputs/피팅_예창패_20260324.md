**한 줄 정의:** 업무 중 PC가 알아서 경력을 쌓고, AI가 이력서용 경력 기록으로 변환해주는 Windows 데스크톱 앱

## 2. 문제 정의

### 창업자가 직접 경험한 구조적 문제

직장인은 일을 하면서도 경력을 쌓지 못한다. 이것은 의지의 문제가 아니라, 구조의 문제다.

이직을 결심한 직장인 3~7년차의 패턴은 공통적이다.

> "어떤 프로젝트를 했지?" → 기억이 없다 → 정리 포기 → 이직 기회 소멸

기억하려면 일을 멈춰야 하고, 일하는 동안에는 기록할 수 없다. 이 구조적 간극이 경력 기록 실패의 본질이다.

**창업자 본인의 경험:** 기획·QA·PM 멀티롤로 실무를 수행하며 이 문제를 반복 경험했다. 경력은 쌓였지만 정리는 되어 있지 않았고, 이직 준비 시 활용 가능한 형태의 기록이 없었다. 이 문제를 해결하기 위해 CareerLog를 기획하고 직접 개발했다.

### 기존 솔루션의 공통 맹점

모든 기존 도구는 "이미 기억하는 것"을 잘 쓰도록 도와줄 뿐이다. 기억 자체를 쌓아주는 서비스는 없다.

| 기존 솔루션 | 방식 | 근본 한계 |
|---|---|---|
| 노션 / 메모 앱 | 직접 입력 | 기록하려면 업무를 멈춰야 함 |
| 링크드인 | 직접 작성 | 과거 회고 방식, 기억 의존 |
| 타임트래킹 (Toggl 등) | 수동 타이머 | 시작/종료 직접 입력 필요 |
| AI 이력서 툴 | 입력 기반 생성 | 입력할 내용이 없으면 무용 |
| **CareerLog** | **PC가 알아서 자동 감지** | — |

## 3. 솔루션 및 혁신성

### "업무 블랙박스" — 일하는 동안 PC가 알아서 경력을 쌓는다

CareerLog는 사용자가 업무 중 의식적으로 무언가를 입력하지 않아도 된다. 사용자가 하는 것은 단 두 가지다.

1. **업무 시작 버튼을 누른다**
2. **이직할 때 복사해서 붙여넣는다**

그 사이의 모든 과정은 CareerLog가 자동으로 처리한다.

```
업무 시작 버튼
    ↓
PC 포그라운드 창 10초마다 자동 감지 (PowerShell + Win32 GetForegroundWindow API)
    ↓
앱별 활동 타임라인 자동 생성
    ↓
업무 종료 버튼
    ↓
Claude AI가 STAR 포맷 경력 기록 자동 생성
    ↓
Markdown / HTML 내보내기, 클립보드 복사
```

### 혁신성 및 차별성

**1. 입력 없는 자동 기록:** 시중의 모든 유사 도구는 사용자의 능동적 입력을 전제로 한다. CareerLog는 PC 활동을 수동 개입 없이 자동 감지한다. 국내외에 이 컨셉의 서비스는 현재 존재하지 않는다.

**2. 로컬 우선 프라이버시 설계:** 활동 데이터는 전량 사용자 PC에만 저장된다. AI 생성 시에도 원본 활동 데이터는 외부로 전송되지 않는다. 직장인이 업무 기록 툴을 기피하는 주요 이유(보안·프라이버시 우려)를 구조적으로 해소한다.

**3. 기술적 안정성:** 외부 네이티브 모듈 없이 Windows Win32 API를 PowerShell을 통해 직접 호출하는 독자적 구현 방식을 채택했다. 이는 패키징·배포 환경에서 발생하는 의존성 오류를 원천 차단한다.

## 4. 시장 분석

### 타겟 시장

**메인 타겟: 이직을 준비하는 직장인 3~7년차**
- 기획자, PM, 디자이너, 개발자 등 PC 기반 전문직
- 경력은 쌓였으나 정리된 기록이 없는 상태
- Windows PC 사용 비중 높고, 로컬 저장 방식에 대한 신뢰도 높음

**세컨더리 타겟:**
- 신입 취준생: 인턴·프로젝트 경험을 포트폴리오로 전환하는 수단
- 프리랜서·N잡러: 다수 프로젝트의 실적을 클라이언트별로 정리

### 시장 규모 (TAM-SAM-SOM)

| 구분 | 규모 | 산출 근거 |
|---|---|---|
| **TAM** | 약 1조원/년 | 국내 사무·전문직 PC 기반 임금근로자 약 840만 명 (통계청) × 월 9,900원 × 12개월 |
| **SAM** | 약 1,583억원/년 | 이직 예비자 + AI 도구 수용 의향자 40% 추정 → 약 132만 명 |
| **SOM** | ARR 6~24억원 | 초기 18개월 내 유료 전환 5,000~20,000명 목표 (베타 데이터 확보 후 갱신) |

### 경쟁 포지셔닝

현존하는 서비스 가운데 "기록 자동화"와 "이력서 품질 출력"을 동시에 제공하는 서비스는 없다. CareerLog는 이 두 축의 교차점을 선점한다.

```
                  이력서 출력 품질 高
                        │
         링크드인        │     CareerLog (목표 포지션)
         잡코리아AI      │
─────────────────────────┼─────────────────────────
 기록 수동              │        기록 자동
─────────────────────────┼─────────────────────────
         노션·메모       │     (아직 존재하지 않는 영역)
                        │
                  이력서 출력 품질 低
```

## 8. 기대 효과 및 사회적 가치

### 직장인의 경력 자본 격차 해소

이직 시장에서 경력 정리의 품질은 현재 개인의 글쓰기 능력과 시간 여유에 따라 결정된다. 업무가 바쁠수록, 글쓰기가 서툴수록, 경력 기록에서 불리해지는 구조다.

CareerLog는 일하는 동안 자동으로 경력을 축적함으로써, 경력 정리에 투입할 시간이 없는 직장인도 동일한 출발선에 설 수 있도록 한다.

- 이직 준비 기간 단축 → 노동 시장 유동성 증가
- AI 기반 경력 기록의 민주화 → 전문직 정보 비대칭 완화
- 로컬 저장 방식으로 직장인 개인정보 보호 구조화

### 시장 확장성

국내 커리어테크 시장은 잡코리아(2021년 카카오 1조 7,900억원 인수), 원티드랩(시총 최대 5,000억원) 등의 사례가 보여주듯 대형 M&A 수요가 검증된 시장이다. CareerLog는 채용 플랫폼이 아직 보유하지 못한 "업무 기록 자동화" 레이어를 선점하는 전략으로, 플랫폼 사업자의 전략적 인수 대상이 될 수 있다.

## 9. 창업 의지

저는 이 문제를 직접 겪었고, 직접 해결했습니다.

기획자·QA·PM으로 일하면서 경력은 쌓였지만 정리된 기록이 없었습니다. 이직을 준비할 때마다 "내가 뭘 했지?"라는 질문 앞에서 막혔고, 그 시간을 반복할 수 없다고 판단해 직접 만들기 시작했습니다.

아이디어에서 출발해 기술 스택 선정, 개발, QA, 배포용 인스톨러 제작까지 1인으로 완성했습니다. 외부 PD와의 협업으로 제품 방향성과 핵심 리스크도 검증했습니다.

예비창업패키지 지원금으로 베타 사용자를 확보하고, 리텐션 데이터를 쌓고, 이 제품이 시장에서 필요한 것임을 증명하겠습니다.
