### 1-3. 단위 경제 및 BEP 검증

| 항목 | 문서 기재값 | 검산 | 판정 |
|------|-----------|------|------|
| 유저당 월 AI 비용 | 8회 x $0.03 = $0.24 (약 320원) | $0.24 x 1,380원(환율) = 331원 -> "약 320원" | 허용 |
| AI 비용 비중 | 6.5% | 320/4,900 = 6.53% | 정확 |
| AI 제외 마진율 | 93.5% | (4,900-320)/4,900 = 93.5% | 정확 |
| 서버 포함 마진율 | 약 69% | (4,900-1,500)/4,900 = 69.4% | 정확 |
| **BEP** | **유료 약 300명** | 300 x 4,900 = 147만원/월 | **오류** (하단 상술) |

**BEP 오류 상세**:
문서에 "서버비 + AI API + 창업자 최소 생활비 포함, 유료 구독자 약 300명 시점"이라 기재. 그러나:
- 서버/인프라: 800만원/12개월 = 월 66.7만원
- AI API: 월 21만원 (MAU 1,000명 기준)
- 창업자 인건비: 월 200만원 (Phase 2 이후 기재)
- 월 고정비 합계: 약 288만원
- BEP = 288만원 / 4,900원 = 약 588명

300명일 때 매출은 147만원으로, 고정비 288만원의 51%에 불과하다. 마진율 69%를 적용해도: 300 x 4,900 x 0.69 = 101.4만원 (변동비 제외 공헌이익)으로 고정비를 커버하지 못한다.

**BEP 300명이 성립하려면**: 창업자 무급 + 서버비 30만원 이하 + AI API 10만원 이하여야 한다. 문서 내 다른 수치들과 모순된다. 이것은 심사위원에게 "재무 이해도 부족"으로 읽힌다.

### 3-2. 리프레이밍 품질 평가

**강점:**
- "이직 도구" -> "매일 쌓이는 커리어 관리 습관 도구"로의 전환이 일관되게 적용됨
- Free 플랜 설계("매일 쌓이는 기록은 무료, 꺼내 쓸 때 유료")가 리프레이밍과 정합
- 설문 결과("가장 끌리는 기능 1위: 업무 활동 자동 기록 47%")가 리프레이밍을 뒷받침

**약점:**
- "귀찮음"이 문서 전반에 과도하게 반복됨 (약 7회). "귀찮음"은 low-severity 페인포인트로 읽힐 수 있다. 심사위원에 따라 "그냥 안 해도 되는 거 아닌가?"라는 반응이 나올 수 있다.
- "귀찮음 -> 경제적 손실"로의 연결고리가 약하다. "경력 미기록으로 인한 연봉 협상 손실"을 정량화하면 문제의 심각성이 훨씬 강해진다.
- 설문에서 "이직 관련 활성 비율 76%"라는 데이터가 각주에 묻혀 있다. 이것을 본문에 전면 배치하면 "이 사람들에게 경력 관리는 귀찮지만 중요한 문제"라는 프레이밍이 강해진다.

## 5. 핵심 피드백 요약

### 강점
1. **제품 완성도가 경쟁자 대비 압도적이다.** v0.8까지 구현 완료, GitHub 배포, Setup.exe 인스톨러, 커밋 이력 공개. 초기창업패키지 지원자 중 이 수준의 완성도를 가진 프로젝트는 10% 이내다.
2. **리프레이밍이 성공적이다.** "이직 도구" -> "매일 커리어가 쌓이는 습관 도구"로의 전환이 일관되게 적용되어 있고, 이 프레이밍이 BM 구조(축적 -> 이탈 비용 -> 유료 전환)와 정합한다.
3. **데이터 기반 의사결정을 시작했다.** WTP 설문 기반 가격 설정(9,900 -> 4,900원), MRR/MAU 산술 교정, 단위 경제 분석 추가. 숫자에 근거를 대려는 자세가 보인다.

### 보완 필수 항목 (불합격 요인 3가지)

1. **[고객 검증] 실사용 데이터 전무 -- 설문 의향과 실제 행동의 갭 미해소**
   - 문제점: 설문 n=34의 "유료 의향 71%"와 실제 결제는 별개다. D30 리텐션 "목표" 40%는 있으나 실측값이 없다. AI 생성 결과물이 시뮬레이션이다.
   - 권장: 2027년 1월까지 베타 30~50명 실사용 (최소 2주), D7/D30 리텐션 실측, 유저 인터뷰 인용구 3~5개, AI 생성 결과 실제 캡처로 교체.
   - 예상 점수 상승: +7~12점

2. **[팀 역량] 1인 체제의 사업화 역량 미검증**
   - 문제점: 3인 심사위원 평균 팀 점수 8.7/15점. 개발은 검증되었으나 마케팅/영업/재무 역량이 보이지 않는다. "탐색적 대화"는 실적이 아니다.
   - 권장: 2027년 1월까지 그로스 마케터 또는 GTM 역량자 1명 합류 확정. 최소한 LOI + 이력 명시.
   - 예상 점수 상승: +5~8점

3. **[자금 계획] BEP 산술 오류 + 예산 배분 비합리성**
   - 문제점: BEP 300명은 산술상 월 147만원으로 고정비(288만원)의 절반 수준. 실제 BEP는 약 550~600명. AI API 1,400만원(실소요 대비 5.6배) 과대 편성. 창업자 인건비 미편성.
