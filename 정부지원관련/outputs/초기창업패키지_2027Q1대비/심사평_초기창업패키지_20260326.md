## 핵심 피드백 요약

### 강점
1. **문제-솔루션 핏이 명확하다.** "귀찮아서 안 한다 -> 버튼 두 개로 자동으로 쌓인다"는 메시지가 직관적이며 설문에서도 뒷받침됨.
2. **실제 작동하는 제품이 있다.** v0.8까지 1인 개발로 구현 완료, GitHub에서 소스 코드와 Setup.exe 확인 가능. 심사위원이 직접 설치해볼 수 있다는 점은 큰 차별점.
3. **기술 구현 수준이 높다.** Win32 API 직접 호출, 앱별 파서 17개, Flow 감지, UIAutomation URL 추출 등 단순 래퍼가 아닌 독자적 기술 구현. 기술 과장이 아님.

### 보완 필수 항목 (불합격 사유 -- 치명적 약점 3가지)

1. **[팀 역량] 1인 체제가 초기창업패키지에서는 치명적**
   - 문제점: 초기창업패키지는 사업자등록 후 실행 중인 "팀"을 평가함. "탐색적 대화 진행 중"은 팀이 아님. 3인 심사위원 모두 팀 항목에서 최하위 점수(8~9점/15점).
   - 권장 수정 방향: 2027년 1월까지 최소 1명의 공동창업자 또는 핵심 팀원이 실제 합류한 상태여야 함. 이상적으로는 그로스/마케팅 역량 보유자. 합류 시점, 역할 분담, 지분 구조가 사업계획서에 명시되어야 함. "채용 예정"이 아닌 "합류 완료"가 필요.

2. **[고객 검증] 실사용 데이터(베타 리텐션)가 전무**
   - 문제점: 설문 n=34는 "사용 의향"일 뿐 "실사용"이 아님. 초기창업패키지는 "제품이 실제로 쓰이고 있는가"를 보는 과제인데, 창업자 본인 외의 실사용 데이터가 0. D30 리텐션, 세션 빈도, AI 생성 횟수 등 핵심 지표가 없음.
   - 권장 수정 방향: 2026년 하반기까지 베타 유저 최소 30명 확보, 2주 이상 실사용 데이터 수집. D7 리텐션, D30 리텐션, 주간 평균 세션 수, AI 생성 횟수, NPS를 측정하여 사업계획서에 포함. 가능하면 유저 인터뷰 인용구 3~5개를 본문에 삽입("이 기능 때문에 계속 쓴다"는 한 줄이 설문 숫자 100개보다 강력).

3. **[시장성] SOM 도달 경로와 수치 일관성 부족**
   - 문제점: MAU 1,000명(12개월 목표)에서 MRR 500만원(유료 약 1,020명 필요)은 전환율 102%로 산술적으로 불가능. SOM 유료 5,000~20,000명 도달 경로에 채널별 CAC/획득 전략이 없음. "블라인드/링크드인 광고"만으로는 구체성 부족.
   - 권장 수정 방향: (1) MAU vs 유료 구독자 수 목표를 재계산 -- MAU 10,000명, 전환율 10%, 유료 1,000명, MRR 490만원 식으로 일관성 확보. (2) 채널별 획득 전략 구체화 -- 블라인드 광고 CPC/전환율 가정, 링크드인 콘텐츠 주간 게시 빈도, SEO 전략, 커뮤니티 운영 계획. (3) CAC(고객획득비용) 추정치 포함 -- 마케팅 예산 2,400만원 / 목표 유저 수로 역산.
