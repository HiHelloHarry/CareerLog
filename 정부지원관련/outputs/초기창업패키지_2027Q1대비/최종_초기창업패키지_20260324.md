## 1. 문제 정의 — 직장인은 귀찮아서 경력 관리를 안 한다

### 핵심 문제: 기록하려면 행동을 멈춰야 한다

직장인 34명(9개 직군) 대상 설문에서 **88%가 경력 정리를 분기 이하 빈도로만 한다**고 응답했다. 이유는 단순하다 — **귀찮기 때문이다.**

업무 중 별도로 기록하는 행위 자체가 부담이기 때문에, 대부분의 직장인은 기록을 시작조차 하지 않는다. 이직을 결심하고 나서야 뒤늦게 경력을 정리하려 하지만, 수년치 기억에 의존해야 하는 상황에서 대부분 포기하거나 직함과 회사명만 남긴 빈 이력서로 마무리된다.

- 이직 결심 후 경력 정리에 상당한 시간이 소요되며, 수치로 된 근거 없이 협상에서 불리해진다
- 노션·옵시디언 같은 도구가 있어도 결국 안 쓰게 되는 이유는 **"귀찮음"** 때문이다
- 결국 문제는 이력서 작성 능력이 아니라, **귀찮아서 기록되지 않은 경험** 그 자체다

### 문제의 규모

- **잠재 고객군**: 통계청 2024년 상반기 지역별고용조사 기준, 경영·회계 관련 사무직 임금근로자 **426만 2천 명** (직업중분류 전체 1위 항목)
- **이직 고려율**: 20~40대 정규직 근로자의 **69.5%**가 이직을 고려 중 (한국경영자총협회 「근로자 이직 트렌드 조사」, 2024, 글로벌리서치 위탁, 전국 1,500명)
- **AI 도구 수용도**: 국내 취업자 업무용 생성형 AI 활용률 **51.8%** — 미국(26.5%)의 약 2배 수준 (한국은행 BOK 이슈노트 제2025-22호, 2025.08)

이직을 고려 중인 사무직만 약 **296만 명 (추정)** 이며, 이 규모의 수요에 대응하는 "경력 자동 축적" 카테고리 서비스는 국내에 존재하지 않는다.

### 기존 솔루션의 구조적 한계

| 기존 솔루션 | 방식 | 한계 |
|---|---|---|
| 노션 / 메모 앱 | 직접 입력 | 기록하려면 행동을 멈춰야 함. 지속 사용률 낮음 |
| 링크드인 | 직접 작성 | 과거 회고 방식, 기억 의존, 업데이트 방치 패턴 |
| 타임트래킹 (Toggl 등) | 수동 타이머 | 시간만 기록, "어떤 성과였는지" 해석 불가 |
| AI 이력서 툴 (Resume.io, Rezi 등) | 입력 기반 생성 | 입력할 내용이 없으면 무용 |
| 사람인 / 잡코리아 | 채용 플랫폼 | 이직 결심 이후에야 사용하는 구조 |

**핵심 공백:** "경력을 자동으로 축적"하는 카테고리 자체가 현재 국내 시장에 없다.

### 타겟 고객

**1순위 — 매일 체계적으로 커리어를 관리하고 싶은 사무직 직장인 (기획자·PM·디자이너·개발자)**
- 경력은 있으나 정리가 안 된 상태로, 이직·연봉 협상·성과 평가 시점에 수치 근거가 없어 불리한 경험이 있는 층
- 지금 당장 이직할 생각은 없지만 "나중에 필요하겠다"는 인식을 가진 층 — 즉각적 이직 도구가 아닌 "매일 쌓이는 경력 보험"으로 포지셔닝
- 설문 결과: 가장 끌리는 기능 1위가 **"업무 활동 자동 기록"(47%)** — 귀찮음 해소에 대한 수요가 가장 높음

**2순위 — 신입 취준생**
- 인턴·프로젝트 경험을 체계적으로 기록하고 싶은 층

**3순위 — 프리랜서·N잡러**
- 다수 클라이언트 프로젝트를 관리하고 포트폴리오를 축적해야 하는 층

**고객 검증 현황 — 설문조사 (n=34, 9개 직군)**

직장인 34명 대상 설문조사를 실시하여 1차 고객 검증 데이터를 확보했다.

| 항목 | 결과 |
|------|------|
| 응답자 규모 | n=34, 9개 직군 (기획·PM·디자이너·개발자·마케터 등) |
| 경력 정리 빈도 | **88%가 분기 이하 빈도** — 이직할 때만 38%, 분기/반기 32%, 거의 안함 9% |
| 가장 끌리는 기능 | 1위: **업무 활동 자동 기록 (47%)**, 2위: AI 경력 생성 (44%), 성과 수치 추출 (44%) |
| 사용 의향 | **76%** (바로 써보고 싶다 29% + 관심 있다 47%) |
| 유료 의향 | **71% (24명)** |
| WTP (지불 의향 가격) | 유료 의향자 중 월 5,000원 이하 **71% (17/24명)** → Pro 가격 4,900원으로 설정 |

> "귀찮아서 관리를 안 한다" → "버튼 두 개로 자동으로 쌓인다" — 이 문제-솔루션 핏이 설문에서 확인됨. 이직 관련 활성 비율 76%(이직 준비 32% + 최근 이직 32% + 시간 없음 12%), 경력 정리에 하루 이상 소요 62%, 수치 표현 어려움 59%로 핵심 페인포인트 검증 완료. 2027년 1월 제출 시점까지 베타 유저 인터뷰를 추가 수행하여 정성 데이터를 보강할 예정.

## 2. 솔루션 — CareerLog: 버튼 두 개로 매일 커리어가 자동으로 쌓이는 도구

### 제품 개요

CareerLog는 Windows 데스크톱 앱(Electron)이다. 업무 시작, 업무 종료 — **두 버튼만으로 매일 커리어가 쌓인다.** STAR 이력서는 필요할 때 꺼내 쓰는 부산물이다.

```
업무 시작 버튼
    ↓
PC 활동 자동 감지 (10초 간격, 앱별 파서 17개 — VS Code·Chrome·Slack·Figma·Notion 등)
    ↓
타임라인 + 커리어 대시보드에 매일 기록이 쌓임
    ↓
업무 종료 버튼
    ↓
필요할 때 → AI가 STAR 포맷 경력 기록 자동 생성
    ↓
Markdown / HTML 내보내기, 클립보드 복사 (이력서·LinkedIn 형식 지원)
```

### 기술 혁신성

**AI 경력 기록 생성 구조:**
- Claude Sonnet API + 독자 설계 STAR 프롬프트 엔지니어링
- 직군·직급·기술스택 프로필을 프롬프트에 주입하여 동일한 활동 데이터에서 직무 맞춤 경력 기록 생성
- 출력 구조: `{situation, task, action, result, skills, metrics_detected, bullets}` — 이력서 직접 활용 가능한 포맷

**v0.7 기록 정확도 고도화:**
- **앱별 창 제목 파서 17개** — VS Code에서 열린 파일명, Chrome에서 방문한 URL, Slack 채널명 등을 구조화 메타데이터(`parsed` 필드)로 저장
- **브라우저 URL 추출** — UIAutomationClient로 Chrome/Edge 주소창을 읽어 도메인·경로 정보 자동 수집 (민감 도메인은 호스트만 저장)
- **키보드+마우스 활동 감지** — `GetAsyncKeyState`(키보드) + `GetCursorPos`/`VK_LBUTTON`(마우스)로 active/light/passive 3단계 분류
- **작업 흐름(Flow) 감지** — 개별 앱 전환을 의미 단위(flow)로 묶어 AI에 구조화 입력. 같은 프로젝트의 다른 파일은 하나의 flow, 5분 이하 이탈은 [보조] 태그

**v0.8 UI 리디자인:**
- **3단 레이아웃** — 프로필 히어로+대시보드 통계 / 월별 캘린더 / 업무 카드 2열 그리드
- **디테일 드로어** — 카드 클릭 시 오른쪽 슬라이드 드로어로 STAR/불릿 상세 표시, 키보드 좌우 이동
- STAR 경력 기록은 드로어에서만 노출 — 일상 화면은 "매일 쌓이는 기록"에 집중

**AI 생성 결과물 품질 예시**

아래는 창업자가 CareerLog를 실사용한 세션에서 생성된 결과물 예시다.

**입력 — 자동 감지된 활동 타임라인:**

| 앱 | 창 제목 | 체류 시간 | 메모 |
|---|---|---|---|
| Figma | 대시보드 v2 — 메인 화면.fig | 2시간 12분 | 대시보드 레이아웃 리뷰 후 피드백 반영 |
| Slack | 디자인팀 채널 | 47분 | — |
| Notion | PRD — 2.0 릴리즈 기획 | 1시간 3분 | — |
| Chrome | Jira - DASH-142 ~ DASH-148 | 28분 | — |
| VS Code | dashboard-api.ts | 15분 | — |

**출력 — AI가 자동 생성한 STAR 경력 기록:**

> **Situation:** 대시보드 v2 메인 화면 UI 개편 프로젝트에서 디자인 리뷰 및 피드백 반영 단계를 담당했다.
>
> **Task:** 디자인팀과 협업하여 대시보드 레이아웃 리뷰를 진행하고, 리뷰 결과를 Figma에 즉시 반영하는 것이 목표였다.
>
> **Action:** Figma에서 대시보드 v2 메인 화면 레이아웃을 2시간 12분간 집중 수정했다. Slack 디자인팀 채널에서 47분간 피드백을 주고받으며 수정 방향을 조율했고, Notion PRD 2.0 릴리즈 기획 문서를 1시간 3분간 업데이트했다. Jira에서 DASH-142~148 티켓 7건의 상태를 확인하고 우선순위를 재조정했다.
>
> **Result:** 대시보드 v2 메인 화면 디자인 리뷰를 완료하고, Jira 티켓 7건의 디자인 반영 사항을 확정했다. 총 4시간 45분의 집중 작업으로 디자인→기획→개발 핸드오프 준비를 완료했다.
>
> **Skills:** Figma, Slack, Notion, Jira, UI/UX 디자인 리뷰, PRD 작성
>
> **이력서용 불릿:**
> - 대시보드 v2 UI 개편 프로젝트에서 Figma 기반 디자인 리뷰 주도, Jira 티켓 7건 반영 사항 확정
> - Slack·Notion 활용 디자인팀-기획팀 간 실시간 피드백 루프 운영, 핸드오프 4시간 45분 내 완료
> - PRD 2.0 릴리즈 기획 문서 업데이트 및 DASH-142~148 우선순위 재조정 수행

메모 한 줄("대시보드 레이아웃 리뷰 후 피드백 반영")이 Context가 되어, 앱 이름과 체류 시간만으로는 불가능한 업무 맥락 복원이 이루어진다. 직군·직급 프로필이 프롬프트에 주입되므로, 동일한 활동 데이터에서도 기획자 관점과 개발자 관점의 경력 기록이 각각 다르게 생성된다.

**프라이버시 보호 설계:**
- 활동 데이터는 사용자 PC 로컬에만 저장, 외부 서버 전송 없음
- AI 경력 기록 생성 시에만 Claude API 호출 (원본 활동 데이터 미전송)
- 온보딩 화면에 "모든 기록은 이 PC에만 저장" 명시

**기술 안정성 검증 이력:**
- UIAutomationClient 방식의 창 제목 오염 버그(타 앱 제목이 교차 기록되는 문제) 발견 및 Win32 GetForegroundWindow() API 직접 호출 방식으로 해결
- 네이티브 모듈(active-win) 없이 PowerShell 영구 프로세스(spawn 1회, 10초 루프)로 Electron 패키징 환경에서의 안정성 확보

### 차별화 포인트

경쟁사들은 모두 "이미 기억하고 있는 것"을 잘 쓰게 도와준다. CareerLog는 **"귀찮음 제로로 매일 커리어가 쌓이는 유일한 자동 기록 서비스"**다.

```
              [기록 자동화 높음]
                    │
                    │   ★ CareerLog
                    │     (귀찮음 제로 — 매일 자동 축적)
                    │
 [커리어 관리 약함] ──────┼────── [커리어 관리 강함]
                    │
        노션·옵시디언│         Resume.io · Rezi
        (수동 기록)  │         (1회성 작성 도움)
                    │
              [기록 자동화 낮음]
```

### 글로벌 경쟁자 비교

| 서비스 | 자동 기록 | AI 경력 변환 | 로컬 저장 | 한국어 지원 | 국내 이직 최적화 | 가격 |
|--------|----------|------------|----------|-----------|---------------|------|

| Rewind AI | O (화면 캡처) | X | O (Mac 전용) | X | X | $19/월 |
| Notion·Obsidian | X (수동 입력) | X | 선택 가능 | O | X | 무료~$8 |
| Resume.io·Rezi | X | O (작성 보조) | X | X | X | $6~$25/월 |

**핵심 차별점:** MS Recall은 화면을 통째로 저장하는 방식으로 개인정보 논란이 지속되고 있으며, 국내 이직 시장에 특화된 AI 경력 기록 변환 기능이 없다. MS가 향후 Recall에 경력 기록 기능을 추가하더라도, 기업 보안 정책에 의한 사용 제한과 로컬 저장 미보장이라는 구조적 약점은 해소되지 않는다. Rewind AI는 Mac 전용이며 경력 문서 생성을 지원하지 않는다. CareerLog는 **귀찮음 제로로 매일 커리어가 쌓이는 유일한 자동 기록 서비스**이며, 앱별 파서 17개와 Flow 감지로 단순 화면 캡처를 넘어 업무 맥락을 구조화하는 유일한 로컬 저장 솔루션이다.

**기술 보호 전략:**
1. **SW 저작권 등록** — CareerLog 소스 코드를 한국저작권위원회에 프로그램 등록 (베타 출시 시점)
2. **영업비밀 관리** — AI STAR 프롬프트 엔지니어링 로직과 활동 병합 알고리즘을 영업비밀로 지정
3. **방어적 공개** — Win32 API 활용 패턴을 기술 블로그에 공개하여 선행기술화, 경쟁자의 특허 선점 차단

### 사회적 가치

**직장인의 협상력 강화**
기록 없이 밀리는 연봉 협상·성과 평가 구조를 개선한다. 근로자가 자신의 기여를 수치로 입증할 수 있을 때 정당한 보상 협상이 가능해진다.

**경력 기록 격차 해소**
기록 능력이 뛰어난 사람뿐만 아니라 기록을 못하는 사람도 자동으로 경력이 쌓이는 구조를 제공한다. 기획자·디자이너·PM처럼 코드로 성과를 증명하기 어려운 직군의 경력 가시화에 기여한다.

**프라이버시 보호 기반 AI 도구 모델 제시**
모든 데이터를 로컬에만 저장하는 구조로, AI 도구 사용에 따른 개인정보 유출 우려 없이 경력 데이터를 안전하게 관리하는 모델을 제시한다.
