## 1. 문제 정의

### 핵심 문제: 직장인은 일하면서 경력을 쌓지 못한다

국내 사무·전문직 직장인의 경력 기록 실패는 의지 문제가 아닌 구조적 문제다.

**이직 결심 후 경력 정리를 포기하는 이유는 단 하나다: "기록하려면 일을 멈춰야 한다."**

- 직장인 3~7년차는 이직 결심 후 평균 2주 안에 경력 정리를 시도하다 포기한다
- "어떤 프로젝트를 했지?" → 기억이 없다 → 포기 → 이직 기회 소멸
- 문제는 경력이 없는 것이 아니라, 경력이 기록되지 않는 것이다

**기존 솔루션의 공통 맹점:**

| 기존 솔루션 | 방식 | 한계 |
|---|---|---|
| 노션 / 메모 앱 | 직접 입력 | 기록하려면 행동을 멈춰야 함 |
| 링크드인 | 직접 작성 | 과거 회고 방식, 기억 의존 |
| 타임트래킹 (Toggl 등) | 수동 타이머 | 시작/종료 직접 입력 필요 |
| AI 이력서 툴 | 입력 기반 생성 | 입력할 내용이 없으면 무용 |

**결론:** 기억 자체를 자동으로 축적하는 서비스는 현재 존재하지 않는다.

### 타겟 고객 및 고객 검증

## 2. 솔루션

### "업무 블랙박스" — PC가 일하는 동안 경력을 자동으로 축적한다

CareerLog는 Windows 데스크톱 앱이다. 사용자가 하는 일은 두 가지뿐이다.

1. 업무 시작 버튼을 누른다
2. 이직할 때 복사해서 붙여넣는다

```
업무 시작 버튼
    ↓
PC 포그라운드 창 10초마다 자동 감지 (PowerShell + Win32 GetForegroundWindow API)
    ↓
앱별 활동 타임라인 자동 생성
    ↓
업무 종료 버튼
    ↓
Claude AI가 STAR 포맷 경력 기록 자동 생성
    ↓
Markdown / HTML 내보내기, 클립보드 복사
```

### 핵심 기능 및 개발 현황 (v0.6 MVP 완성)

| 기능 | 상태 |
|---|---|
| PC 활동 자동 감지 (10초 폴링, Idle 15분 감지) | 완료 |
| 세션 시작/종료, 프로젝트 태깅, 작업 유형 선택(8개 카테고리) | 완료 |
| 타임라인 뷰 (앱별 그룹 / 시간순 토글) | 완료 |
| 커리어 대시보드 (히트맵, 스트릭, 인사이트 카드) | 완료 |
| AI STAR 경력 기록 자동 생성 (Claude API) | 완료 |
| 직군·직급·스킬 프로필 AI 프롬프트 주입 | 완료 |
| Markdown/HTML 내보내기, 클립보드 복사 | 완료 |
| Windows x64 Setup.exe 인스톨러 | 완료 |

### 기술 차별화 및 혁신성

**[수정 항목 — Before]**
> "Claude Sonnet API + 독자 설계 STAR 프롬프트 엔지니어링"만 언급. 설계 내용 없음.

**[수정 항목 — After]**

**AI 적용 방식: 맥락 인식 STAR 프롬프트 설계**

단순히 Claude API를 호출하는 것이 아닌, PC 활동 데이터를 이력서 관점으로 재해석하는 프롬프트 설계가 핵심 기술이다.

**입력 구조:**
```
[사용자 프로필] 직군: PM | 직급: 5년차 시니어 | 기술스택: Figma, Jira, SQL
[세션 데이터]  총 3시간 22분 | 프로젝트: 결제 개선 프로젝트
[앱 타임라인]  Figma(1h 20m) → Slack(45m) → Notion(35m) → Zoom(42m)
[작업 유형]    기획 / 커뮤니케이션 (사용자 선택)
```

**출력 구조:** `{situation, task, action, result, skills, metrics_detected, bullets}`
- `situation`: 프로젝트 태그 + 앱 패턴으로 업무 맥락 자동 추론
- `task`: 직군·직급 기준 책임 영역 기술 (PM: "기획 및 의사결정 주도", 개발자: "기술 구현 및 문제 해결")
- `action`: 앱 타임라인 기반 능동형 동사 서술 (Figma → "화면 설계", Jira → "이슈 관리", Zoom → "이해관계자 회의 진행")
- `bullets`: 이력서 직접 붙여넣기 가능한 1~3줄 성과 요약

**개인화 핵심:** 동일한 활동 로그도 직군에 따라 다른 관점의 경력 기록 생성. PM과 개발자가 같은 Figma 사용 시간을 기록해도, PM에게는 "UI 기획 주도", 개발자에게는 "디자인 검토 참여"로 서술이 달라진다.

**오류 방지 설계:** 활동 로그 5분 미만 또는 단일 앱 사용 세션은 AI 생성 비활성화 → 할루시네이션 방지 및 데이터 품질 보장

**프라이버시 보호 설계:**
- 활동 데이터는 사용자 PC 로컬에만 저장 (외부 서버 전송 없음)
- AI 생성 시에만 Claude API 호출 (활동 원본 데이터 미전송, 요약 메타데이터만 전달)

**기술 안정성 검증:**
- UIAutomationClient 방식의 창 제목 오염 버그 → Win32 GetForegroundWindow() API 직접 호출로 해결
- 네이티브 모듈(active-win) 의존 없이 PowerShell 영구 프로세스 방식으로 패키징 안정성 확보
