# CareerLog MVP 마일스톤

> 기준일: 2026.03.17
> 개발 방식: Claude Code + 변지현 (기획/PM/디렉팅)
> 플랫폼: Electron 트레이 앱 (Windows / Mac)
> 목표: 업무 자동 감지 → 경력 기록 자동화

---

## 제품 핵심 플로우

```
트레이 아이콘 상주
       ↓
[업무 시작] 클릭
       ↓
백그라운드에서 자동 감지 (30초마다)
  - 활성 앱명
  - 문서/창 제목
  - 체류 시간
       ↓
[업무 종료] 클릭
       ↓
타임라인 화면 팝업
  10:00  Figma │ 온보딩_UX_v3.fig       1h 20m  [+ 메모]
  11:20  Slack │ #product-team          15m     [+ 메모]
  11:35  PPT   │ 2분기 기획서_최종.pptx  45m     [+ 메모]
  12:20  Zoom  │ 주간 팀 미팅           1h      [+ 메모]
       ↓
사용자가 각 항목에 디테일 입력
  "온보딩 3개 화면 신규 디자인, 팀 리뷰 반영"
       ↓
Claude가 경력 기록으로 변환·저장
```

---

## 기술 스택

| 영역 | 기술 | 이유 |
|------|------|------|
| 앱 프레임워크 | Electron | 트레이 상주, Win/Mac 지원 |
| UI | React + Tailwind | 빠른 개발, 깔끔한 UI |
| 활성 창 감지 | `active-win` (npm) | 앱명·문서명 자동 수집 |
| 로컬 저장 | SQLite (`better-sqlite3`) | 빠름, 파일 기반 |
| AI 분석 | Claude API | 활동 → 경력 기술로 변환 |
| 빌드/배포 | electron-builder | .exe / .dmg 패키징 |

---

## 확장성 설계 (처음부터 열어둘 것)

```
integrations/
  ├─ slack.js      ← 채널 활동, 멘션, DM 요약
  ├─ calendar.js   ← Google/Outlook 미팅 자동 수집
  ├─ github.js     ← 커밋, PR, 코드리뷰
  └─ notion.js     ← 문서 편집 활동
```

MVP에서는 구현 안 하지만 **데이터 모델에 source 필드**를 처음부터 넣어서 나중에 연동 시 기존 데이터와 호환되게 설계.

## Phase 1 — 트레이 앱 기반 구축 (4월, 4주)

**1주차: 프로젝트 세팅**
- [ ] Electron + React 프로젝트 생성
- [ ] 트레이 아이콘 등록 + 메뉴 ([업무 시작] / [업무 종료] / 종료)
- [ ] `active-win`으로 30초마다 활성 창 감지
- [ ] SQLite에 원시 활동 로그 저장

**2주차: 타임라인 UI**
- [ ] [업무 종료] 클릭 시 팝업 창
- [ ] 타임라인 뷰 (시간대별 앱·문서·체류시간)
- [ ] 짧은 활동(5분 미만) 자동 병합
- [ ] 각 항목에 메모 입력 필드

**3주차: AI 분석**
- [ ] Claude API 연동
- [ ] 타임라인 + 메모 → 경력 기록 자동 생성
- [ ] 생성된 경력 기록 확인·수정 화면
- [ ] 로컬 SQLite에 경력 기록 저장

**4주차: 안정화**
- [ ] 로그인 시 자동 시작 옵션
- [ ] 날짜별 기록 히스토리 조회

**완료 기준:** 변지현 본인이 일주일 이상 실제 업무에 사용
